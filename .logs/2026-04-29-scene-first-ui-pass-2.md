# Scene-first UI Pass 2

## 1. 변경 요약

- 의뢰 화면을 현재 메인 목표, 첫 30분 성장 루트, 진행 의뢰 3개, 다음 잠금 의뢰로 재구성했다.
- 장비 화면에 현재 배/수레, 능력치, 다음 추천 성장, 새로 유리한 루트를 먼저 보이게 했다.
- 장부/도감 화면을 방문, 거래, 발견, 장부 조각, 최근 거래가 모이는 성취 화면으로 재구성했다.
- 의뢰, 장비, 장부에서 시장/지도/장비/의뢰로 이어지는 바로가기 버튼을 강화했다.
- 1차 pass의 어두운 반투명 패널, 황동 라인, 청록 CTA, 장면 우선 구성을 이어받았다.

## 2. 의뢰 화면 개선 내용

- 상단에 현재 메인 목표와 진행도 바를 배치했다.
- 목표 단계는 상품/지도/장비 아이콘으로 표시한다.
- 첫 30분 성장 루트는 노드형 길로 유지하되 의뢰 화면의 중심으로 올렸다.
- 의뢰 카드는 출처 아이콘, objective 아이콘 단계, 보상 칩, 바로가기 버튼으로 압축했다.
- 잠긴 의뢰는 별도 작은 카드로 분리해 조건만 짧게 보여준다.

## 3. 장비 화면 개선 내용

- 현재 상단 장비 카드에서 배와 수레를 함께 보여준다.
- 다음 추천 성장 카드에서 부족한 돈과 새로 유리한 루트를 강조한다.
- 개인 장비, 동료, 배/수레 구매 목록은 하위 영역으로 밀어 구매 목록 느낌을 줄였다.
- 모바일 가로 844x390에서는 구매 목록보다 현재 장비와 다음 목표가 먼저 보이도록 조정했다.

## 4. 장부/도감 화면 개선 내용

- 상단 요약에 방문 항구, 거래 상품, 완료 의뢰, 총 손익, 만난 사람 수를 도장형 칩으로 표시한다.
- 보유 화물은 판매 추천 버튼과 함께 성취 화면 안에 통합했다.
- 도감 진행은 항구, 상품, 발견, 장부 조각 진행률 칩으로 표시한다.
- 최근 거래는 8개까지만 우선 보이고, 아이콘과 손익 중심으로 압축했다.
- 다음 완성할 장부 또는 다음 발견 목표를 별도 추천 카드로 보여준다.

## 5. 화면 간 연결 강화

- 의뢰 objective의 목적지가 지도인 경우 가능한 route를 선택하고 지도 탭으로 이동한다.
- 의뢰에서 필요한 시장 행동은 시장 탭으로 이어진다.
- 장비 화면에서 열린 길 보기 버튼으로 지도 탭에 연결한다.
- 장부/도감 화면에서 지도 보기, 시장 보기, 의뢰 보기 버튼으로 루프를 연결한다.
- 화물 판매 추천은 기존 cargo sale hint와 지도 route 선택 흐름을 유지한다.

## 6. 1차 디자인 시스템과 통일한 요소

- compact HUD와 floating top tabs 구조를 그대로 유지했다.
- 어두운 먹색/남색 반투명 hero panel, 황동 라인, 청록 primary CTA를 사용했다.
- 정보는 긴 문단보다 아이콘, 칩, 짧은 버튼으로 압축했다.
- scene-first 느낌을 위해 의뢰/장비/장부 main panel에도 배경 이미지와 어두운 overlay를 깔았다.

## 9. 남은 한계

- 의뢰 화면의 성장 루트는 SVG/HTML 기반이며, 전용 픽셀아트 지도형 배경으로 더 고도화할 수 있다.
- 장비 구매 목록은 그룹화되었지만 완전한 상점 NPC 장면까지는 아니다.
- 장부/도감은 성취형으로 정리했지만, 도감 항목별 상세 팝업은 아직 없다.

## 10. 다음 추천 작업

1. 의뢰 성장 루트를 전용 픽셀아트 길 배경으로 제작하고 요정 말풍선을 더 작게 통합한다.
2. 장비 구매를 조선소/수레방 NPC 장면과 연결해 구매 전후 비교 연출을 강화한다.
3. 도감 항목 클릭 시 발견 카드, 항구 위치, 거래한 상품 기록을 작은 팝업으로 보여준다.
