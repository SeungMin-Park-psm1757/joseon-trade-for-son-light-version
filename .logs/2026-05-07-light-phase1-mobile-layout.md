# Light Phase 1 Mobile Layout

Date: 2026-05-07

## 기존 문제 목록

- 시작 화면에 빈 공간이 많아 게임 첫인상이 약했다.
- 도시/지도/장터/가게 화면의 이동 방식이 서로 달라 화면들이 따로 노는 느낌이 있었다.
- 요정 말풍선과 목표 스트립이 항상 큰 공간을 차지해 메인 콘텐츠가 눌렸다.
- 장터 화면은 하단 행동 버튼과 공통 이동이 겹치면 판매 카드가 답답하게 잘렸다.
- 모바일 세로에서 PC 패널을 줄인 듯한 비율이 일부 남아 있었다.

## 새 레이아웃 원칙

- portrait first: 360x800, 390x844, 412x915, 430x932 기준.
- 공통 골격: 상단 상태바, 전면 메인 화면, 공통 하단 네비, 작은 요정 힌트, 목표 스트립.
- 메인 화면은 도시/지도/장터/이동/가게 중 현재 행동 하나가 크게 보이게 한다.
- 주요 버튼은 56px 안팎의 터치 크기를 유지한다.
- 하단 네비는 도시, 지도, 장터, 가게 4개로 제한한다.
- 튜토리얼에 필요한 경우만 장터 내부의 기존 이동 버튼을 보여준다.

## 구현 내용

- 시작 화면에 라이트 모드 타이틀, 게임 이름, 한 줄 목표 문구를 추가했다.
- 공통 `QuickNav`를 추가해 도시/지도/장터/가게 이동을 통일했다.
- 상단 상태바 높이와 pill 크기를 모바일 기준으로 재정리했다.
- 요정 말풍선을 더 작고 가벼운 힌트 영역으로 줄였다.
- 목표 스트립 높이를 줄여 메인 콘텐츠 공간을 확보했다.
- 장터 자유 플레이에서는 중복 하단 버튼을 숨기고 공통 네비를 사용한다.
- 공통 디자인 토큰(`--junior-gap`, `--junior-radius`, `--junior-touch` 등)을 추가했다.

## Viewport 검수 결과

- 360x800: 도시 화면이 잘리지 않고 4개 주요 행동과 하단 네비가 보인다.
- 390x844: 시작/도시/지도/장터/가게 화면 모두 주요 버튼이 터치 가능한 크기로 보인다.
- 412x915: 도시 화면의 배경, 액션 카드, 말풍선 여백이 안정적이다.
- 430x932: Galaxy S26 Ultra급 세로 화면에서 상단바와 하단 네비가 과도하게 커지지 않는다.
- 844x390: 참고용 landscape에서도 지도/패널/요정/네비가 겹치지 않는다.

## 남은 한계

- 가게 화면은 기능적으로 맞지만 카드가 아직 UI 카드 목록 느낌이 강하다.
- 장터 화면은 상품 수가 늘면 세로 스크롤이 필요하므로 다음 단계에서 카테고리/짐 영역을 더 게임답게 나눌 수 있다.
- 21개 도시별 완전 고유 배경 일러스트는 별도 아트 패스가 필요하다.
