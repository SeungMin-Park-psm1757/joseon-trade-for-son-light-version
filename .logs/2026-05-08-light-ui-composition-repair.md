# 2026-05-08 Light UI Composition Repair

## 1. 변경 요약
- 라이트버전의 기능 추가가 아니라 화면 교정에 집중했다.
- 지도, 도시 도착 컷씬, 퀴즈/이벤트 화면, 요정 말풍선 배치의 시각 규칙을 하나의 모바일 우선 레이아웃으로 정리했다.
- 우측 고정 요정 패널이 본문 공간을 침범하지 않도록 데스크톱/가로 화면에서도 단일 컬럼 플레이 구조로 고정했다.

## 2. 지도 화면 수정
- 해상 점선과 육로 선을 더 얇고 반투명하게 낮춰 지도 배경보다 튀지 않도록 조정했다.
- 도시 노드는 크기, 외곽선, 선택 상태, 잠금 상태를 통일했다.
- `책 안동`, `항구 울산`처럼 라벨 앞에 분류 단어가 붙는 방식을 줄이고, 작은 배지/아이콘과 도시명 중심으로 정리했다.
- 도시 선택 카드가 지도 중앙을 크게 가리지 않도록 작게 줄이고, 모바일에서는 보조 문구를 숨겨 도시명/특산품/출발/도시로만 보이게 했다.
- 선택 도시에는 확대와 링 강조가 들어가 클릭 피드백이 더 분명하게 보이도록 했다.

## 3. 컷씬 화면 수정
- 도착 컷씬을 배경 일러스트, 정우, 요정, 하단 대화 카드의 4층 구조로 재정리했다.
- 요정은 안내자 역할에 맞게 정우보다 작게 배치했다.
- 정우는 장면 하단 쪽에 위치하도록 스케일과 위치를 조정했다.
- 대화는 우측 별도 패널 대신 하단 대화 카드 안에서 정돈되게 보이도록 했다.

## 4. 퀴즈/이벤트 화면 수정
- 퀴즈 화면을 장면 영역, 문제 카드, 선택지, 피드백 흐름으로 통일했다.
- 방패 같은 아이콘만 뜨는 빈 화면 느낌을 줄이기 위해 결과 화면도 카드형 영역과 메시지/버튼 중심으로 보이도록 수정했다.
- 의미가 불명확했던 큰 원형 기호는 `맞춤말`, `날씨`, `도움`, `이야기`처럼 짧은 역할 라벨로 바꿨다.
- 선택지 버튼의 크기와 간격을 맞춰 모바일에서 누르기 쉽게 했다.

## 5. 제거한 문제 요소
- 지도 위를 덮는 두껍고 과한 점선 항로
- 지도 중앙을 크게 가리는 도시 선택 카드
- 도시명 앞에 분류 단어가 붙는 지저분한 라벨
- 데스크톱/가로 화면의 우측 고정 요정 패널
- 컷씬에서 정우보다 크게 보이는 요정 배치
- 빈 화면처럼 보이는 퀴즈 결과 화면
- 엔딩 이미지가 다시 하기 버튼 클릭을 가로막는 포인터 이벤트 문제

## 6. 검수 결과
- `cd junior-game && npm run build`: 통과
- `cd junior-game && npm run test`: 60개 테스트 통과
