# PLAN.md — 팔도상단: 조선의 바람 1차 프로토타입

## 1. Player goal

플레이어는 조선 후기 평민 상인으로 부산포에서 시작한다. 상품을 사고팔고, 땅길과 바닷길을 이동하며, 위험 이벤트를 넘기고 돈을 모아 더 좋은 배와 수레를 산다. 1차 프로토타입의 장기 목표는 제주 항로를 안정적으로 오가고, 왜관 허가장을 얻어 대마도 첫 교역을 시도하는 것이다.

## 2. Main loop

```text
상태 확인
→ 현재 항구의 시장 가격 확인
→ 상품 구매
→ 지도에서 이동 가능 루트 선택
→ 이동 소요일/위험 확인
→ 이동 중 이벤트 또는 전투 처리
→ 도착지에서 판매
→ 손익 확인
→ 자동 저장
→ 장비 구매 또는 다음 교역 준비
```

## 3. Inputs and controls

모바일 세로 화면 기준입니다.

- 탭: 항구, 상품, 이동지, 하단 탭 선택
- 버튼: 구매, 판매, 이동, 이벤트 선택, 어업, 저장/이어하기
- 수량 조절: `-` / `+` 버튼 또는 간단 숫자 입력
- 긴 설명은 접기/펼치기 또는 카드 상세로 처리

## 4. Win and fail states

### MVP success states

- 첫 거래로 이익을 냄
- 손수레 또는 새 배 구매
- 제주 항로 성공
- 왜관 허가장 획득
- 대마도 첫 교역 성공

### Failure states

완전 게임오버는 만들지 않습니다. 대신 손실이 있습니다.

- 도적/해적 패배: 돈, 화물, 내구도, 사기 손실
- 태풍/암초: 선박 내구도 손상
- 갯벌 좌초: 시간 손실 또는 선원 피로
- 배/수레 상실 가능. 단, 회복 의뢰와 어업으로 재기 가능

## 5. Progression and difficulty

- 시작: 작은 돛단배, 지게, 적은 자금
- 초반: 부산포–대구–전주–목포 단거리 교역
- 중반: 여수–통영–제주 항로
- 후반: 왜관 허가장 후 대마도 교역
- 월별 가격 변동으로 매달 다른 장사 판단을 유도
- 위험은 존재하지만 아이가 계속 이어갈 수 있게 회복 루트를 제공

## 6. Visual direction

- 전통 지도 느낌의 따뜻한 배경색
- 지도는 실제 지도를 정밀 재현하지 않고, 한국 남부·중부·제주·대마도 위치관계를 읽기 쉬운 추상 2D로 표현
- 버튼은 크고 명확하게
- 상태바에는 돈, 날짜, 위치, 월간 사건을 항상 표시
- 외부 에셋이 없어도 SVG, CSS, 이모지, 단순 아이콘으로 플레이 가능
- 무료 2D 에셋은 후속 단계에서 적용

## 7. Stack and hosting assumptions

- Vite + React + TypeScript
- CSS/SVG 지도
- LocalStorage 자동 저장
- 정적 웹앱 배포 가능: Vercel, Netlify, GitHub Pages 등
- 서버 없음
- 모바일 웹 중심, PC에서도 폭 480px 앱 컨테이너로 표시

## 8. Milestone order

### Milestone 0 — 준비

- 데이터 JSON 검증
- 타입 정의 보강
- 폴더 구조 정리
- `npm run build` 기본 통과

### Milestone 1 — 저장과 기본 상태

- 새 게임/이어하기
- 부산포 시작
- LocalStorage 자동 저장
- 상단 상태바

### Milestone 2 — 지도와 이동

- 항구 버튼 표시
- 현재 위치 강조
- 연결 루트만 표시
- 이동 시 날짜 증가
- 대마도 허가장 제한

### Milestone 3 — 시장과 장부

- 현재 항구 가격 표시
- 구매/판매
- 화물칸 제한
- 최근 거래 로그
- 손익 피드백

### Milestone 4 — 월별 경제

- 월별 가격표 생성
- 같은 달 가격 유지
- 월 변경 시 가격 갱신
- 월간 사건 표시

### Milestone 5 — 이벤트와 간단 전투

- 이동 위험 이벤트
- 선택지 효과 반영
- 도적/해적 자동 또는 선택지 전투
- 패배 후 복구 가능

### Milestone 6 — 어업과 장비

- 어업 버튼과 보상
- 배/수레 구매
- 장비 능력치가 화물칸·이동에 반영

### Milestone 7 — 퀘스트와 도감

- 튜토리얼 퀘스트 3개 이상
- 진행/완료 저장
- 도시·상품 도감 기본 화면

### Milestone 8 — 모바일 브라우저 검증

- 390px 폭에서 가로 스크롤 없음
- 주요 버튼 44px 이상
- 하단 탭과 이벤트 모달 안정화
- `tests/ACCEPTANCE_CHECKLIST.md` 기준 자체 점검

## 9. MVP route for manual playtest

```text
1. 새 게임 시작
2. 부산포 시장에서 면포 또는 건어물 구매
3. 대구 또는 통영으로 이동
4. 이벤트 발생 여부 확인
5. 도착지에서 판매
6. 손익 로그 확인
7. 새로고침 후 이어하기 확인
8. 손수레 또는 어선 구매 목표 확인
9. 제주 루트 위험 정보 확인
10. 대마도 루트는 허가장 전 제한되는지 확인
```

## 10. First implementation boundary

첫 구현에서 완성해야 할 것은 “재미있는 교역 루프”입니다. 그래픽 완성도, 조선풍 정교 에셋, 실시간 전투, 도시 투자, 중국·일본 본토 확장은 후순위입니다.

## 11. 2026-04-28 current plan correction

프로젝트는 초기 세로형 MVP에서 가로형 모바일 픽셀아트 웹게임으로 방향이 바뀌었습니다. 이후 계획은 다음 기준을 우선합니다.

- 기본 viewport: 844x390, 932x430.
- 세로 화면: 가로 플레이 안내 표시.
- 표현 방향: 항구 배경, 장면 핫스팟, 장터 진열대, NPC 말풍선/간판, 한국 지도 이동, 배/수레 성장, 경량 사운드.
- 핵심 루프는 유지: 항구 -> 시설 -> 시장 구매/판매 -> 지도 이동 -> 이벤트/결과 -> 장부/의뢰/성장.
- JSON 데이터와 localStorage 저장을 유지합니다.
- 이번 단계에서도 서버 저장, 중국/일본 본토 확장, 항구 투자, 실시간 전투는 넣지 않습니다.
- 향후 작업 큐는 `docs/NEXT_WORK_ROADMAP.md`를 기준으로 합니다.
- 현재 압축 컨텍스트는 `docs/CURRENT_CONTEXT.md`를 기준으로 합니다.

## 12. 2026-04-28 Market Hotspot Tools Companion Checkpoint

이번 체크포인트는 장면형 UI 이후의 시장/항구/장비/동료 보강 단계입니다.

- 시장 상품은 정렬된 진열대처럼 보이고 터치하기 쉬워야 한다.
- 판매는 평균 매입가, 현재 판매가, 가까운 추천 판매처를 함께 보여준다.
- 항구 핫스팟은 시장은 좌상단/마을 쪽, 술집은 시장 근처, 관청은 언덕/정자 쪽, 조선소는 아래 물가 쪽으로 읽혀야 한다.
- 의뢰는 한 번에 약 3개만 노출해 다음 행동을 흐리지 않는다.
- 배/수레는 탈것 성장이고, 칼/어업도구/측량도구/거래장부/언어도구는 개인 장비 성장이다.
- 튜토리얼은 시간여행한 정우를 돕는 요정 안내 형식이다.
- 동료와 가족 도움은 가벼운 함대 시스템으로 두고, 함대 이름을 저장한다.
- 다음 후속은 주요 항구별 개별 핫스팟 좌표와 동료별 사건/시설 반응 대사다.

## 13. 2026-04-29 Sale Guide Companion Equipment Checkpoint

이번 체크포인트는 "물건을 샀는데 어디에 팔지 모르겠다"를 줄이는 판매 안내와, 동료/장비 능력치가 실제 선택지에 보이는 것을 목표로 한다.

- 지도에는 현재 화물 기준 최고 판매처를 아이콘 말풍선과 강조 루트로 표시한다.
- 화물/장부 화면에는 보유 물건별 판매 추천 버튼을 먼저 보여주고, 누르면 지도에서 해당 루트가 선택된다.
- 동료와 개인 장비 능력치는 이벤트 선택지의 호위/기술 판정 문구에 반영한다.
- 개인 장비는 1단계와 2단계 티어를 갖고, 이전 장비가 있어야 다음 장비를 살 수 있다.
- 장비 탭은 개인 장비를 먼저 보여주고, 배/수레는 탈것 성장으로 분리한다.
- 다음 패스는 항구별 고유 NPC 반응, 동료별 사건 대사, 장비 구매 축하 연출을 묶어 진행한다.

## 14. 2026-04-29 Port Companion Purchase Moment Checkpoint

이번 체크포인트는 동료와 구매 보상이 화면에 더 직접적으로 드러나게 만드는 단계입니다.

- 시설 화면에는 합류한 동료가 짧게 조언하는 줄을 추가한다.

- 개인 장비 구매도 배/수레처럼 축하 모달을 띄우고, 능력치와 다음 행동을 보여준다.
- 동료 영입도 같은 축하 모달을 사용해 역할과 능력치를 보여준다.
- 함대 패널에는 현재 합류한 동료 한 명의 한마디를 띠 형태로 표시한다.
- 다음 패스는 항구별 고유 반응을 데이터화하고, 장비 구매 후 실제로 새로 유리해진 루트를 지도에 더 선명하게 표시하는 작업이다.

## 15. 2026-04-29 Economy Systems Checkpoint

이번 체크포인트는 교역 게임의 신뢰도를 무너뜨리던 같은 항구 즉시 매수/매도 차익 문제를 막고, 가격을 플레이어가 읽고 계획할 수 있는 구조로 재정비하는 단계입니다.

- 상품 데이터에는 변동성, 공급 수준, 수요 수준, 생산지, 세율, 티어/희귀도를 명시한다.
- 가격은 기본가에 생산지/수요지, 항구 성향, 월별 사건, 계절, 운송 위험, 지역 특성을 곱해 월별로 계산한다.
- 매수가와 매도가를 분리하고 세금/상인 마진/스프레드를 반영해 같은 항구 즉시 되팔기는 항상 손해가 나게 한다.
- 추천 패널과 상품 상세는 실제 quote 계산값을 사용해 `어디서 얼마에 사고 어디서 얼마에 팔면 얼마가 남는지`를 보여준다.
- 시장 진열은 해당 항구의 기본 판매품뿐 아니라 플레이어가 들고 온 판매 가능 화물과 판매 추천 품목을 우선 포함한다.
- 상단 명성 축약어는 상인/탐방/호위 풀네임과 툴팁으로 의미를 설명한다.
- 기존 저장 슬롯은 가격 모델 버전이 다르면 월별 가격표를 재생성해 이전 가격표가 새 경제 모델을 우회하지 못하게 한다.

## 16. 2026-04-29 Pixel Art Style Checkpoint

이번 체크포인트는 임시 PNG처럼 보이던 상품/이동/안내자 표현을 같은 픽셀아트 해상감으로 묶고, 시장/항구/지도/장비 화면의 정보 위계를 한 단계 더 읽기 쉽게 만드는 단계입니다.

- 전 상품 아이콘은 런타임 SVG 픽셀 프레임으로 교체해 48px 기준 외곽선, 제한 팔레트, 카테고리별 실루엣을 통일한다.
- 요정은 조선 세계관의 시간여행 길잡이/수호령처럼 보이는 픽셀 캐릭터로 바꾸고 기본/기쁨/경고 표정을 상황에 따라 쓴다.
- 튜토리얼 코치와 시장 상품 상세에는 요정 말풍선형 조언을 붙여 생산지, 수요지, 손해 거래 같은 핵심 순간을 설명한다.
- 이동 중 배경이 비치던 PNG 차량/배 토큰은 투명 배경의 픽셀 SVG 토큰으로 교체한다.
- 상품 타일, 거래 팝오버, 추천 패널, 장비 카드의 테두리/그림자/배경을 같은 한지+목재 계열 픽셀 UI로 정리한다.
- 1365x768과 1920x1080 스크린샷, 844x390 스모크/비주얼 테스트로 겹침, 잘림, 가로 스크롤, 이동 토큰 배경을 검수한다.

## 17. 2026-04-29 Full Korea Map Navigation Checkpoint

이번 체크포인트는 기존 남부 중심의 둥근 섬처럼 보이던 지도를 북부까지 포함한 한반도 전도형 교역망으로 고치고, 도시 선택과 우측 이동 패널의 정합성을 안정화하는 단계입니다.

- 지도 SVG는 북부 지역, 중부, 남부, 제주, 울릉도, 대마도 위치가 한눈에 읽히는 한반도 실루엣으로 다시 그린다.
- 북부 거점은 평양, 남포, 신의주, 원산, 함흥, 청진, 개성을 추가하고 기존 남부/도서 좌표도 새 지도에 맞게 보정한다.
- 북부 루트는 한양-개성-평양-신의주/남포, 평양-원산, 남포-강화, 강릉-원산-함흥-청진으로 연결한다.
- 노드 hit target을 키우고 `data-testid`를 부여해 hover/click/selected 상태가 작은 점이나 겹친 라벨에 흔들리지 않게 한다.
- 우측 route preview에는 목적지 타입, 수리 상태, 적재 상태, 예상 수익을 고정 그리드로 표시한다.
- 1365x768과 1920x1080에서 남부 도시 선택, 북부 도시 선택, 제주/대마도/흑산도 노드가 하단 코치에 가리지 않는지를 Playwright로 확인한다.

## 18. 2026-04-29 Fishing Minigame Narrative Audio Checkpoint

이번 체크포인트는 어업을 단일 보상 버튼에서 작은 모험형 미니게임으로 바꾸고, 요정 안내와 오디오 런타임을 함께 보강하는 단계입니다.

- 항구 어업은 `어장 선택 -> 조류 타이밍 선택 -> 그물 회수 선택 -> 결과 기록` 4단계로 진행한다.
- 어장마다 위험도, 수확 기대치, 선호 타이밍, 적합한 회수법, 보상 상품군을 분리한다.
- 결과는 어업 실력, 배 fishingBonus, 동료/장비 fishing 스탯, 월간 fishingBonus, 물때, 피로, 어장 위험, 선택 정확도를 합산해 성공/보통/실패로 나뉜다.
- 요정은 첫 어업, 좋은 어장, 거친 바다 경고, 타이밍 성공, 실패/빈 그물, 대어/고수익 결과에 따라 다른 말과 표정을 보여준다.
- 어업 결과는 기존 `ledger.fishing`에 품질, 어장, 위험 설명을 함께 기록하고, 의뢰 진행도 `fishingCount`와 자동 저장 흐름을 유지한다.
- 오디오는 BGM 교차 페이드, SFX 채널 제한, 동일 효과음 dedupe, SFX 우선순위, BGM ducking을 적용해 겹침 시 끊김을 줄인다.
- 844x390, 1365x768, 1920x1080에서 어업 준비/결과 모달의 CTA 노출, 코치 패널 겹침, 가로 오버플로를 Playwright 스크린샷으로 확인한다.
