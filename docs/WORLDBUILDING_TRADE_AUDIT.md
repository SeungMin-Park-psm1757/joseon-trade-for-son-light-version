# 세계관/지도/교역 감사: 조선 후기 북방 장시 패스

## 최종 시대 설정안

- 기준 시대: 19세기 조선 후기, 개항 이전부터 초기 개항기 직전의 분위기.
- 조선의 중국 방면 상대: 명나라가 아니라 청.
- 북방 방면 상대: 러시아 제국 세력이 두만강 너머에 닿기 시작한 후반기 분위기. 단, 본토 확장이 아니라 경흥 근처 국경 장시 소문과 제한 교역으로만 표현한다.
- 일본 방면: 본토 확장이 아니라 부산포 왜관과 대마도까지만 유지한다.
- 신의주는 근대 철도/교량 이후 성장한 도시성이 강하므로 데이터 ID `sinuiju`는 저장 호환성 때문에 유지하되, 게임 표시명은 `의주`로 바꾼다.
- 청진은 1908년 개항장 성격이 강하므로 데이터 ID `chongjin`은 유지하되, 게임 표시명은 두만강 변경 거점인 `경흥`으로 바꾼다.

## 지도 수정 요약

- 지도 표시명은 조선 후기 분위기에 맞춰 한양, 부산포, 제포, 영일만, 의주, 경흥을 사용한다.
- 한반도 북부 지도에 압록강과 두만강 라벨을 추가했다.
- 두만강은 엷은 점선 강줄기로 표시해 국경 감각을 만든다.
- `qing_yalu_market`과 `tumen_north_market`은 한반도 바깥쪽에 아주 가까운 국경 장시로 배치했다.
- 루트는 기존 `land/sea` 타입을 유지하면서, `terrain: ["border_trade"]` 태그로 국경 교역로를 구분한다.
- 국경 교역로는 보라색 점선으로 표시되어 일반 땅길/바닷길과 다르게 읽힌다.

## 데이터 설계 메모

- 저장 호환성을 위해 기존 ID `sinuiju`, `chongjin`은 바꾸지 않았다.
- 새 국경 장시 ID 2개를 추가했으므로 발견 카드와 항구 풍미 데이터도 함께 추가했다.
- 해외 교역은 본토 확장이 아니라 국경 접촉면이다.
- 허가 없이 이동할 수 없도록 route `permitRequired`를 부여했다.
- 허가장은 신규 의뢰 `qing_border_pass_intro`, `tumen_trade_pass_intro`에서 받을 수 있도록 했다.

## 검증 포인트

- `ports.json`의 모든 거점은 발견 카드와 풍미 데이터를 가진다.
- `routes.json`의 국경 루트는 `land`를 유지해 기존 이동/적재 로직을 깨지 않는다.
- `audit:consistency`에서 루트/상품/발견/거울 데이터 정합성이 통과해야 한다.
