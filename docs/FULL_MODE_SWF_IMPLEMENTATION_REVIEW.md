# Full Mode SWF Implementation Review

Last updated: 2026-05-13

이 문서는 `reference-swf/analysis/full-mode/` 분석 결과가 현재 full mode 구현에 실제로 반영되었는지 검수한 결과다. SWF의 자산, 코드, 문구, UI 그래픽은 사용하지 않았고, 게임 구조와 리듬만 조선 후기 무역 RPG에 맞게 재해석했다.

## 1. 구현 반영표

| 분석된 SWF 요소 | 반영 여부 | 반영 방식 | 미반영 이유 | 품질 평가 | 후속 작업 |
|---|---|---|---|---|---|
| 빠른 버튼 피드백 | 반영 | 구매/판매/이동/이벤트 선택 후 토스트, 플로팅 텍스트, 결과 모달 표시 | 없음 | 좋음 | 버튼별 SFX와 시각 피드백을 더 세분화 |
| 항구별 cheap/high goods | 반영 | 특산품/수요품/월별 유행품 칩, 가격 이유, 추천 판매처 표시 | 없음 | 좋음 | 두 번째 거래 추천을 더 구체화 |
| 화물칸 제약 | 기존+보강 | 시장 구매 차단, 지도 출발 준비표, 배/수레별 짐칸 표시 | 없음 | 좋음 | 손수레 이후 큰 화물 상품의 체감 강화 |
| 지도 이동 전 위험 preview | 반영 | `route-risk-card`, 위험 태그, 준비물, 추천 상품, 보상 힌트 | 없음 | 좋음 | 위험 대비 보유 장비 체크를 더 명확히 |
| 이동 중 긴장감 | 반영 | 첫 이동 사건 보장, 이후 루트 위험도 기반 사건 발생 | 없음 | 좋음 | 긴 플레이에서 빈도 로그 수집 |
| 부분 실패와 회복 | 반영 | 통행세, 우회, 기다림, 구조, 협상, 수리비/내구도 손실 | 없음 | 좋음 | 패배 후 회복 의뢰를 추가 |
| 도적/해적/검문/날씨 사건 | 반영 | 도적, 해적, 태풍, 갯벌, 암초, 검문, 길손 상인, 표류선 구조 | 없음 | 좋음 | 사건별 장면 이미지 품질 보강 |
| 전투 선택 | 부분 반영 | 도적/해적은 맞서기, 도주, 지불, 우회 선택과 자동 결과 | 실시간/복잡 전투는 범위 초과 | 충분 | 간단 턴제 화면은 후속 |
| 보상 연출 | 반영 | 의뢰 완료 모달, 사건 결과 칩, 발견 카드, 장부 조각, 토스트 | 없음 | 좋음 | 장부 조각 완성 연출 강화 |
| 업그레이드 성취감 | 반영 | 장비 구매 모달, 새로 유리해지는 길, 부족 금액 표시 | 없음 | 보통~좋음 | 의뢰 완료 모달과 장비 모달 순서 연출 개선 |
| 반복 루프 명확성 | 반영 | 모든 핵심 화면에 현재 목표/다음 추천 스트립 추가 | 없음 | 좋음 | 다음 추천을 더 상황별로 세분화 |
| 첫 30분 루프 | 반영 | 부산포 면포 → 대구 → 첫 사건 → 판매 → 첫 의뢰 → 손수레 목표 | 없음 | 좋음 | 두 번째 거래 안내 보강 |
| 항구 신뢰/장부 목표 | 반영 | 발견 보상, 항구 신뢰, 장부 조각 카드, 장기 목표 노출 | 없음 | 보통 | 후반 장부 보상과 엔딩 연결 |
| 장비 ladder | 반영 | 손수레, 짐말, 어선, 연안 상선, 무장 상선 등 역할/가격/힌트 | 없음 | 보통~좋음 | 실제 구매 시점 2시간 로그 검증 |
| 동료/도구 선택지 지원 | 기존+보강 | 이벤트 선택지에 스킬/동료/도구 보정과 대응 준비물 표시 | 없음 | 좋음 | 동료별 전용 사건 대사 추가 |
| 짧은 퀘스트 단계 | 반영 | 첫 장사, 소금 장사, 손수레, 남해, 제주, 왜관 목표 | 없음 | 좋음 | 목표 잠금/해금 설명을 데이터화 |
| dense Flash window UI | 미반영 | 장면 중심 UI 유지 | 현재 프로젝트 방향과 충돌 | 올바른 제외 | 없음 |
| SWF 원 자산/코드/문구 | 미반영 | 사용하지 않음 | 저작권 및 프로젝트 정체성 | 올바른 제외 | 계속 금지 |
| 카리브 해적 판타지 | 미반영 | 남해 해적/왜관/수군/검문으로 변환 | 조선 후기 세계관과 맞지 않음 | 올바른 제외 | 없음 |
| 복잡한 실시간 해전 | 미반영 | 선택형/자동 결과로 제한 | MVP 범위와 모바일 UI에 부담 | 올바른 제외 | 후속 확장 후보 |

## 2. 핵심 항목별 판정

### 빠른 버튼 피드백

구매, 판매, 이동, 사건 선택, 의뢰 완료, 장비 목표까지 짧은 피드백이 있다. SWF식 즉각 반응은 구조적으로 반영되었다.

### 이동 중 긴장감

첫 이동은 `길손 상인`으로 손실 없는 사건을 보장한다. 이후에는 루트 위험도, 사건 덱, 월별 위험 보정으로 도적/해적/태풍/검문이 발생한다. 첫 길이 비어 있지 않아 초반 리듬이 좋아졌다.

### 보상 연출

첫 판매 후 돈, 상인 신용, 의뢰 완료 모달, 다음 목표가 이어진다. 사건 결과도 결과 칩과 다음 추천으로 닫힌다.

### 업그레이드 성취감

손수레와 배 구매 목표가 장비 화면에서 잘 보인다. QA 결과 손수레 420냥은 너무 빨라 480냥으로 조정했다. 조정 후 첫 장사 뒤 `22냥 부족`으로 목표가 살아난다.

### 반복 루프

현재 목표 스트립, 시장 추천, 지도 위험 카드, 장부 목표가 반복 루프를 연결한다. 다만 두 번째 거래 추천은 더 직접적인 문구가 필요하다.

### 튜토리얼 흐름

첫 장사 완료는 5~8분 안에 가능하다. 첫 이동 사건, 첫 판매, 첫 의뢰, 손수레 목표가 이어져 SWF식 초반 hook은 충분히 반영되었다.

## 3. 품질 등급

| 영역 | 등급 | 이유 |
|---|---|---|
| 초반 명확성 | A- | 첫 목표와 첫 장사가 뚜렷함 |
| 교역 판단 | A- | 가격/평균/추천/되팔기 손익이 보임 |
| 이동 긴장감 | B+ | 첫 사건과 위험 카드가 좋음. 긴 플레이 빈도는 추가 검증 필요 |
| 위험 이벤트 | B+ | 결과 칩과 대응 정보는 좋음. 전용 장면 품질은 후속 |
| 장비 성장 | B | 목표가 보임. 손수레 가격 조정 완료, 중반 배 곡선은 추가 검증 필요 |
| 장기 목표 | B | 장부/허가 목표가 보이지만 엔딩 연결은 후속 |
| 모바일 안정성 | A- | 844x390, 932x430에서 가로 스크롤 없음 |

## 4. 후속 작업

1. 두 번째 거래 추천을 루트/상품 단위로 더 강하게 표시한다.
2. 장비 구매 후 의뢰 완료 모달과 장비 축하 모달이 순서대로 자연스럽게 보이도록 연출 우선순위를 다듬는다.
3. 도적/해적/태풍/검문 전용 장면 이미지를 보강한다.
4. 2시간 자동 시뮬레이션으로 이벤트 빈도와 평균 손실을 확인한다.
5. 제주/대마도/북방 장부 목표를 엔딩 진행률과 더 직접 연결한다.
