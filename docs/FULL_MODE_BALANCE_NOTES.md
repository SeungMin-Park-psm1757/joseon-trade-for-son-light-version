## 1. QA 결론

| 질문 | 판정 | 근거 |
|---|---|---|
| 첫 10분 동안 무엇을 해야 할지 명확한가? | 통과 | 시작 목표, 시장 추천, 지도 루트, 첫 이동 사건, 판매 목표가 한 흐름으로 이어진다. |
| 첫 장사 성공이 5~10분 안에 가능한가? | 통과 | 자동 플레이 기준 3.1초, 실제 플레이 환산 5~8분. 부산포 면포 2개 구매, 대구 이동, 판매, 첫 의뢰 완료까지 확인했다. |
| 첫 30분 안에 장비 목표가 보이는가? | 통과 | 첫 장사 후 장비 화면에서 손수레가 `22냥 부족`으로 표시된다. 목표는 보이되 즉시 구매는 막았다. |
| 지역을 이동할 이유가 분명한가? | 통과 | 면포는 부산포 생산/대구 판매, 소금과 건어물은 내륙 판매 추천으로 연결된다. |
| 특산품/수요품 차이가 실제로 느껴지는가? | 통과 | 시장 간판, 상품 칩, 거래 팝업의 가격 이유와 추천 판매처로 노출된다. |
| 이동 이벤트가 너무 많거나 적지 않은가? | 대체로 통과 | 첫 이동은 의도적으로 1회 보장한다. 이후는 루트 위험도와 월별 위험을 따르며 짧은 루트는 과다하지 않다. |
| 해적/도적 이벤트가 짜증이 아니라 긴장감이 되는가? | 통과 | 대응 준비물, 선택지 힌트, 결과 칩, 다음 추천이 있어 손실이 설명된다. |
| 장비 가격이 너무 싸거나 비싸지 않은가? | 조정 후 통과 | 손수레 420냥은 첫 장사 직후 바로 구매되어 빠르다고 판단, 480냥으로 조정했다. |
| 저장/이어하기가 안정적인가? | 통과 | 이벤트 후, 장비 목표 확인 후, 새로고침 이어하기에서 위치/돈/수레/장부가 유지된다. |
| SWF식 보상 리듬이 반영되었는가? | 통과 | 구매/판매 토스트, 사건 결과 칩, 의뢰 완료 모달, 장비 목표 안내, 다음 추천 행동이 짧게 이어진다. |

## 2. 플레이 시뮬레이션

### 시나리오 A: 초반 장사

| 항목 | 결과 |
|---|---|
| 흐름 | 부산포 시작 → 면포 2개 구매 → 대구 선택 → 길손 상인 사건 → 대구 도착 → 면포 2개 판매 → 첫 장사 완료 |
| 목표 시간 | 5~10분 |
| 실제 판정 | 통과. 실제 플레이 환산 5~8분. |
| 결과 상태 | 위치 `daegu`, 돈 `458냥`, 완료 의뢰 `tutorial_first_trade`, 거래 기록 4건, 이동 기록 1건 |
| 감각 평가 | 첫 성공까지 너무 오래 걸리지 않고, 이동 중 사건이 첫 길을 비우지 않는다. |

### 시나리오 B: 장비 성장

| 항목 | 결과 |
|---|---|
| 흐름 | 첫 장사 4개 거래 기록 후 장비 화면 확인 |
| 조정 전 | 손수레 420냥, 첫 장사 직후 458냥이라 즉시 구매 가능 |
| 문제 | 장비 성취가 너무 빨라 두 번째 거래 동기가 약해짐 |
| 조정 | 손수레 가격을 480냥으로 변경 |
| 조정 후 | 첫 장사 직후 `22냥 부족` 표시. 두 번째 짧은 거래가 자연스럽게 필요 |
| 목표 시간 | 15~25분 |
| 실제 판정 | 조정 후 통과 예상. 첫 장사 후 목표 노출, 1회 추가 거래 후 구매 가능 곡선 |

### 시나리오 C: 해상 진출

| 항목 | 결과 |
|---|---|
| 흐름 | 대구 → 부산포 복귀 → 진해(제포) 해상 루트 → 장비 목표 확인 |
| 목표 시간 | 30~45분 |
| 실제 판정 | 대체로 통과. 남해 항구 방문과 배 정박지 이동이 정상 기록됨 |
| 확인 상태 | 방문 항구 `busanpo`, `daegu`, `jinhae_jepo`; 현재 배 `small_sailboat`; 배 정박지 `jinhae_jepo` |
| 한계 | 제주/대마도 준비는 보이지만, 30~45분 내 실제 제주 진입은 아직 중반 목표에 가깝다. 의도에 맞는다. |

### 시나리오 D: 위험 이벤트

| 이벤트 | 검수 결과 |
|---|---|
| 도적 | 대응 정보 1개, 선택지 3개, 결과 칩 6개, 다음 추천 표시 |
| 해적 | 대응 정보 1개, 선택지 3개, 결과 칩 3개, 다음 추천 표시 |
| 태풍 | 대응 정보 1개, 선택지 2개, 결과 칩 3개, 다음 추천 표시 |
| 검문 | 대응 정보 1개, 선택지 2개, 결과 칩 2개, 다음 추천 표시 |

위험 이벤트는 전부 `event-intel-row`, 선택지, 결과 칩, `event-next-hint`가 표시된다. 육로/해로 손상 대상도 기존 로직대로 분리된다.

## 3. 가격/보상/장비 밸런스 조정

| 항목 | 이전 | 변경 | 이유 |
|---|---:|---:|---|
| 손수레 가격 | 420냥 | 480냥 | 첫 장사 직후 즉시 구매되어 성장 목표가 너무 빨리 소모됨. 첫 장사 후 22냥 부족하게 만들어 두 번째 거래 동기를 만든다. |

유지한 값:

- 시작 돈 300냥은 유지한다. 면포 2개 구매가 가능하고, 초반 실수에도 완전 막힘이 없다.
- 첫 장사 보상 80냥은 유지한다. 첫 성공의 보상감이 충분하다.
- 어선 2200냥, 연안 상선 5800냥은 유지한다. 30~45분에는 목표가 보이고, 실제 구매는 중기 루프에 남는다.

## 4. 이벤트 빈도 조정

이번 패스에서 이벤트 확률 공식은 바꾸지 않았다.

| 구간 | 판정 |
|---|---|
| 첫 이동 | 1회 보장. 초반 재미와 학습을 위해 유지 |
| 짧은 육로 | 위험 2, 3일 기준 체감 가능한 빈도 |
| 짧은 해로 | 해적/구조/암초가 가끔 나오되 매번 나오지는 않음 |
| 고위험 해로 | 제주/흑산도/외해는 장비와 수리비 준비가 필요 |

후속 밸런스에서는 실제 2시간 플레이 로그로 `baseEventChancePerTravelDay`와 `riskLevelMultiplier`를 다시 보정한다.

## 6. 남은 문제

- 첫 장사 뒤 두 번째 거래의 최적 루트를 더 노골적으로 추천할 수 있다.
- 해상 진출은 목표가 보이지만, 제주/대마도 준비까지의 중간 보상이 더 필요하다.
- 이벤트 빈도는 자동 QA 기준으로 안정적이나, 긴 플레이에서 연속 손실이 쌓이는지는 추가 로그가 필요하다.
- 장비 구매 후 의뢰 완료 모달과 장비 축하 모달이 순서대로 뜨는 경우가 있어, 후속 UI 패스에서 우선순위 연출을 더 세련되게 만들 수 있다.

## 7. 다음 추천 작업

1. 첫 장사 완료 직후 `다음 추천: 대구에서 삼베/약초 사서 부산포 팔기`처럼 두 번째 거래 추천을 더 구체화한다.
2. 손수레 구매 후 남해 루트로 가기 전 `건어물 4개`, `수리비 50냥` 같은 준비 체크리스트를 만든다.
3. 2시간 자동 시뮬레이션으로 평균 돈, 평균 손실, 이벤트 빈도, 장비 구매 시점을 기록한다.
4. 제주/대마도 목표는 실제 입장보다 준비 과정 보상을 늘린다.
