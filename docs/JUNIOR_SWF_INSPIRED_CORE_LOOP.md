# Junior SWF-Inspired Core Loop

Last updated: 2026-05-14

목적: full mode의 SWF-inspired 루프를 초1-초2도 플레이할 수 있는 junior 핵심 루프로 재정의한다. Junior mode는 작은 무역 RPG이며, full mode의 복잡한 장부/위험/장비 수치를 직접 가져오지 않는다.

## 1. 핵심 문장

정우는 도시에서 물건을 사고, 길을 따라 다른 도시로 가고, 작은 사건을 지나, 물건을 팔아 돈과 별과 도시 도장을 모은다.

## 2. Junior 루프 요약

```text
도시 도착
-> 오늘 할 일 확인
-> 장터에서 물건 고르기
-> 짐에 싣기
-> 지도에서 도시 고르기
-> 길 카드 확인
-> 이동 장면
-> 작은 사건 또는 퀴즈
-> 도착
-> 물건 팔기
-> 돈/별/도장 보상
-> 수레/배 목표 확인
-> 300냥과 꼬마 거상 배지로 이어짐
```

## 3. 단계별 설계

| 단계 | 화면 | 버튼 | 요정 문구 | 데이터 변화 | 보상 |
|---|---|---|---|---|---|
| 1. 도시 도착 | `JuniorCityScreen` | `장터 가기`, `지도 보기`, `탈것 보기` | "부산 장터에서 시작하자." | `currentCityId`, `visitedCityIds` 유지 또는 첫 방문 추가 | 첫 방문이면 도시 도장 후보 |
| 2. 요정이 오늘 할 일을 알려줌 | 도시 화면의 `오늘 할 일` 카드 | 목표 버튼 1개 | "대구에서 면포를 팔아보자." | `tutorialStage` 또는 파생 목표 갱신 | 길을 잃지 않는 안내 |
| 3. 장터에서 물건 선택 | `JuniorMarketScreen` | 상품 카드 | "이 도시는 면포가 좋아." | 선택 전 변화 없음 | 상품 이해 |
| 4. 산 값 확인 | 상품 카드 내부 | `짐에 싣기` 또는 상품 카드 클릭 | "산 값은 10냥이야." | 선택 전 변화 없음 | 가격 읽기 |
| 5. 짐에 싣기 | 장터 화면, 짐칸 표시 | `짐에 싣기` | "면포를 실었어. 대구에서 인기 많아." | `coins - price`, `cargo +1`, `marketPressure.buy +1`, `message` 갱신 | 구매 피드백, 짐칸 튐 |
| 6. 지도에서 도시 선택 | `JuniorMapScreen` | 도시 버튼 또는 `출발` | "초록 도시로 갈 수 있어." | `destinationCityId` 후보 선택 | 목적지 감각 |
| 7. 길 카드 확인 | 지도 선택 카드 | `출발하기` | "산길이야. 천천히 가자." | 출발 전 변화 없음 | 이동 전 기대감 |
| 8. 이동 장면 | `JuniorTravelScreen` | `도착하기` 또는 자동 후 버튼 | "수레가 고개를 넘어가." | `currentStep='travel'` | 5초 안팎의 여정감 |
| 9. 사건/퀴즈 가능 | `JuniorEventScreen` | 2지선다 또는 선택지 1-3개 | "바른 말을 골라보자." | `selectedEventId`, `seenEventIds`, `quizWrongStreak` | 정답이면 별/돈, 오답이어도 작은 손실 |
| 10. 도착 | `visitIntro` 또는 도시 화면 | `둘러보기`, `장터 가기` | "대구에 도착했어." | `currentCityId=destination`, `visitedCityIds + city` | 첫 방문 도시 도장 |
| 11. 도시 소개 또는 장터 | 첫 방문 컷 또는 도시 화면 | `장터 가기` | "대구는 약초와 장터가 유명해." | `tutorialStage` 증가 | 지역 기억 |
| 12. 물건 판매 | 장터의 내 짐 영역 | `팔기` | "면포를 팔아보자." | `coins + sellPrice`, `cargo -1`, `marketPressure.sell +1` | 돈 증가, 짐칸 비움 |
| 13. 돈/별 보상 | 결과 말풍선 또는 카드 | `계속하기` | "잘했어! 돈이 늘었어." | 좋은 판매지는 `stars +1`, `badges` 후보 | `+냥`, `별 +1` |
| 14. 수레/배 목표 안내 | 도시 또는 상점 화면 | `탈것 보기` | "손수레까지 40냥 남았어." | 상태 변화 없음 또는 구매 시 `vehicleId`/`boatId` 변경 | 짐칸 +1, 바닷길 자신감 |
| 15. 도시 도장/엔딩 목표 업데이트 | 도시 하단 목표 또는 도장 카드 | `계속 장사`, `집으로 가기` | "도장 3개를 모아보자." | `badges`, `completedEnding`, `completedRuns` | 도시 도장, 꼬마 거상 배지, 300냥 엔딩 |

## 4. 첫 플레이 예시

| 시간 | 흐름 | 아이가 느껴야 하는 것 |
|---|---|---|
| 0-30초 | 시작, 부산 도시 화면, 오늘 할 일 표시 | "장터를 누르면 되는구나." |
| 30초-1분 | 면포 카드 확인, 산 값 보기, 짐에 싣기 | "돈을 내고 물건을 샀다." |
| 1-2분 | 지도에서 대구 선택, 길 카드 확인 | "다른 도시로 간다." |
| 2-3분 | 수레 이동, 쉬운 맞춤법 퀴즈 | "길에서 일이 생겼다." |
| 3-5분 | 대구 도착, 면포 판매, 돈/별 보상 | "팔면 돈이 늘어난다." |
| 5-8분 | 손수레 목표와 두 번째 장사 추천 | "조금 더 하면 새 수레를 산다." |

## 5. 화면별 문구 원칙

| 화면 | 좋은 문구 | 피할 문구 |
|---|---|---|
| 오늘 할 일 | "대구에서 면포를 팔아보자." | "수요지가 높은 상품을 선택하여 차익을 실현하세요." |
| 시장 | "여기서 사기 좋아." | "평균가 대비 매입 스프레드가 낮음." |
| 지도 | "산길이야. 천천히 가자." | "위험도 3, 소요일 2, 이벤트 확률 7%." |
| 사건 | "바른 말을 골라보자." | "도적 이벤트에 대한 대응 전략을 선택하세요." |
| 결과 | "+15냥, 별 +1" | "상인 신용 +1, 항구 신뢰 +2, 내구 -3." |
| 장비 | "짐을 하나 더 실어." | "적재량 +1, 이동 안정성 +4%." |
| 장기 목표 | "도장 3개를 모아보자." | "남해 장부 조각 2/5, 허가 조건 미달." |

## 6. 데이터 변화 기준

| 데이터 | Junior에서의 의미 | 사용 방식 |
|---|---|---|
| `coins` | 살 돈, 팔아서 늘어나는 돈, 300냥 엔딩 목표 | 결과 카드에 크게 표시 |
| `stars` | 좋은 판매, 친절한 선택, 퀴즈 성공의 기분 보상 | 별 아이콘으로 표시 |
| `cargo` | 지금 들고 있는 물건 | 그림 짐칸으로 표시 |
| `cargoLimit` | 실을 수 있는 수 | `2칸`, `3칸`처럼 단순 표시 |
| `vehicleId` | 수레 성장 | 손수레/큰 수레 목표 |
| `boatId` | 바닷길 성장 | 작은 나룻배/돛배 목표 |
| `visitedCityIds` | 도시 도장 | 도장 수와 도장판 |
| `badges` | 장기 칭찬 | 2개 이하만 노출, 전체는 도장/배지 화면 후보 |
| `seenEventIds` | 반복 퀴즈 방지 | 내부 처리 |
| `quizWrongStreak` | 힌트가 필요한지 판단 | 오답 2회 이상이면 요정 힌트 |
| `marketPressure` | 같은 물건을 많이 사고팔 때 가격 살짝 변화 | UI에는 "오늘은 값이 조금 바뀌었어" 정도만 |

## 7. 사건과 손실 기준

| 상황 | Junior 결과 |
|---|---|
| 퀴즈 정답 | `별 +1`, 작은 돈 보상, "무사히 지나갔어." |
| 퀴즈 오답 | `돈 -1~3` 또는 `짐 하나 잃음` 중 아주 작은 손실 |
| 좋은 사건 | `돈 +`, `별 +`, 이야기 단서 |
| 이야기 사건 | 도시/설화 한 문장과 도장/단서 |
| 위험 사건 | 겁주는 표현 없이 "길이 조금 막혔어." 정도 |
| 큰 실패 | 만들지 않는다. 항상 계속할 수 있어야 한다. |

## 8. 성장 루프

| 목표 | 열리는 재미 | 표시 방식 |
|---|---|---|
| 손수레 | 짐칸 3칸 | "짐을 하나 더 실어." |
| 큰 수레 | 짐칸 4칸 | "먼 장터도 든든해." |
| 장사 수레 | 짐칸 5칸 | "큰 장사 준비 끝." |
| 작은 나룻배 | 바닷길 출발 가능 | "바닷길을 건널 수 있어." |
| 작은 돛배 | 먼 바닷길 자신감 | "제주 가기 쉬워." |
| 도시 도장 3개 | 초반 수집 목표 | 도장판 첫 줄 완성 |
| 300냥 | 엔딩 선택 | "집으로 돌아가 자랑하기." |

## 9. 구현 시 첫 경계

첫 구현은 새 게임 시스템을 추가하기보다 현재 `juniorFlow.ts`와 `JuniorApp.tsx`에서 이미 존재하는 흐름을 더 읽기 쉽게 만드는 방향이 좋다.

| 먼저 할 것 | 이유 |
|---|---|
| 오늘 할 일 파생 함수 | 저장 구조 변경 없이 목표 안내를 붙일 수 있다. |
| 지도 길 카드 문구 정리 | 기존 `JUNIOR_ROUTES`의 `terrain`, `fairyText`, `arrivalHint`를 활용할 수 있다. |
| 시장 카드 문구 정리 | 기존 가격 함수와 도시 상품 목록을 그대로 쓴다. |
| 결과 카드 보상 칩 | 기존 `JuniorReward`와 `eventResultText`를 시각적으로 묶으면 된다. |
| 도장/장비 목표 노출 | 기존 `visitedCityIds`, `coins`, `vehicleId`, `boatId`를 활용한다. |
