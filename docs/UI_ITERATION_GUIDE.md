# 모바일 UI 반복 개선 가이드

## 1. 기본 원칙

UI 개선은 작은 단위로 처리합니다. 기능이 이미 작동하는 앱에서 광범위한 리디자인을 시도하면 저장, 이동, 시장 로직이 깨질 수 있습니다.

```text
하나의 화면 + 하나의 문제 + 하나의 검증 기준
```

예시:

- 좋은 요청: “390px 폭에서 시장 상품 카드의 구매/판매 버튼 간격을 8px 늘리고, 버튼 높이를 44px 이상으로 맞춰줘.”
- 나쁜 요청: “전체 UI를 예쁘게 바꿔줘.”

## 2. UI surface 목록

| Surface | 예시 개선 |
|---|---|
| 시작 화면 | 새 게임/이어하기 버튼 크기, 설명 문구 간소화 |
| 상태바 | 돈/날짜/위치 가독성, sticky 처리 |
| 지도 | 항구 버튼 크기, 현재 위치 강조, 루트 라벨 |
| 항구 패널 | 시설/위험 태그 배지, 설명 접기 |
| 시장 | 가격표 정렬, 구매/판매 수량 컨트롤 |
| 화물/장부 | 화물칸 사용량, 최근 손익 표시 |
| 배·수레 | 구매 가능/불가 상태, 현재 장비 강조 |
| 이벤트 모달 | 선택지 버튼, 손실/보상 시각화 |
| 하단 탭 | 탭 높이, safe-area, 활성 탭 강조 |

## 3. 권장 모바일 기준

- 기준 폭: 390px
- 최소 지원 폭: 360px
- 앱 최대 폭: 480px
- 버튼 최소 높이: 44px
- 본문 최소 글자: 14px
- 주요 숫자: 16px 이상, 굵게
- bottom safe-area 반영
- 가로 스크롤 금지

## 4. Codex UI 수정 절차

1. 현재 앱을 실행한다.
2. 수정 대상 route/surface를 연다.
3. 모바일 viewport에서 현재 문제를 확인한다.
4. 필요한 파일만 최소 수정한다.
5. 같은 화면과 viewport에서 결과를 확인한다.
6. 변경 파일과 확인 결과를 보고한다.

## 5. UI 수정 중 지켜야 할 것

- LocalStorage 저장 키 변경 금지
- `GameState` 구조 변경 금지. 필요한 경우 별도 승인
- 데이터 ID 변경 금지
- 교역/이동/월별 경제 로직 임의 변경 금지
- 스타일 수정을 위해 이벤트/시장 기능을 제거하지 말 것
