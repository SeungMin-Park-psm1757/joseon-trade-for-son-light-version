# Junior SWF Redesign Spec

Last updated: 2026-05-14

목적: full mode에서 검증된 목표, 추천, 위험, 결과, 성장 리듬을 junior-game에 맞게 재설계한다. 이 문서는 구현 전 GDD/UX 기준 문서이며, SWF의 이미지, 사운드, 코드, 문구, 캐릭터, UI 그래픽을 복사하지 않는다.

## 1. 설계 방향

Junior-game의 새 목표는 다음 한 문장으로 정의한다.

> 정우가 조선 장터를 돌아다니며 특산품을 사고팔고, 길 위 사건을 풀고, 돈과 별을 모아 수레와 배를 장만하고, 300냥을 모아 집으로 돌아가는 초등 저학년용 작은 무역 RPG.

핵심 경험은 `내가 고른다 -> 내가 산다 -> 내가 이동한다 -> 내가 사건을 푼다 -> 내가 판다 -> 내가 돈을 모은다 -> 내가 성장한다 -> 내가 엔딩을 본다`다.

기존 junior-game의 장점은 유지한다.

| 유지할 장점 | 유지 방식 |
|---|---|
| 독립 앱 구조 | `junior-game/`와 `joseon_trade_junior_save_v1` 유지 |
| 전면 화면형 흐름 | 도시, 장터, 지도, 이동, 사건, 탈것, 엔딩 화면 유지 |
| 지역 특산품 장사 | `buyGoodIds`, `sellGoodIds`, 가격 함수 유지 |
| 맞춤법/선택 퀴즈 | 작은 사건으로 유지하되 결과 피드백 강화 |
| 수레/배 성장 | `JUNIOR_VEHICLES`, `JUNIOR_BOATS`를 주요 목표로 유지 |
| 300냥 엔딩 | money goal을 유지하되 도장/배지와 연결 |

새로 선명하게 할 리듬은 다음과 같다.

| 리듬 | Junior 변환 |
|---|---|
| 목표 | `오늘 할 일` 한 문장과 버튼 하나 |
| 추천 | 장터/지도에서 자연어 힌트 |
| 위험 | 숫자 없는 `길 카드`와 작은 사건 |
| 결과 | `+냥`, `별 +1`, `짐 하나 잃음`, `괜찮아` |
| 성장 | 손수레/배/도장/300냥 목표 |

## 2. 화면별 설계

### A. 시작/이어하기

| 항목 | 설계 |
|---|---|
| 역할 | 새 플레이와 이어하기를 분명히 고르게 한다. |
| 주인공 | 정우와 요정 바람이를 첫 화면의 감정 축으로 둔다. |
| 이어하기 카드 | 현재 도시, 돈, 별, 최근 목표를 표시한다. |
| 처음 하기 | 저장이 없으면 `처음 하기`, 저장이 있으면 `이어하기`를 기본 버튼으로 둔다. |
| 오늘 목표 미리보기 | "부산 장터에서 첫 물건을 사자." 같은 한 문장. |
| 모바일 최적화 | 세로 390px에서 버튼 56px 이상, safe area 여백 유지. |
| PWA | 설치/오프라인 안내는 보조로만 두고 첫 행동을 가리지 않는다. |

권장 화면 구조:

| 영역 | 내용 |
|---|---|
| 상단 | 게임 제목, 작은 장부 또는 정우 그림 |
| 중앙 | 정우와 요정, 한 문장 목표 |
| 카드 | 이어하기 정보: 도시, 돈, 별 |
| 하단 | `이어하기` 또는 `처음 하기` 큰 버튼 |

### B. 도시 화면

| 항목 | 설계 |
|---|---|
| 역할 | 모든 루프의 허브. 아이가 다음 행동을 고르는 곳이다. |
| 도시명 | `부산 장터`, `대구 장터`처럼 도시와 장터 감각을 함께 표시. |
| 도시 배경 | 현재 `backgroundAsset`/`scene`을 사용하고, 텍스트보다 장면이 먼저 보이게 한다. |
| 이곳 특산품 2개 | `buyGoodIds` 앞 2개를 `이 도시 물건` 칩으로 보여준다. |
| 오늘 할 일 카드 | 화면 상단 또는 중앙에 가장 강한 카드로 둔다. |
| 주요 버튼 | `장터`, `지도`, `짐`, `탈것`. 초반 튜토리얼은 한 버튼만 강조. |
| 도시 도장 | 첫 방문 도장은 도시명 옆 작은 도장 또는 카드에 표시. |
| 다음 목표 카드 | 손수레, 작은 배, 300냥, 도장 3개 중 하나만 보여준다. |

오늘 할 일 예시:

| 상황 | 문구 | 강조 버튼 |
|---|---|---|
| 새 게임 직후 | "부산 장터를 둘러보자." | 장터 |
| 화물이 없음 | "이 도시 물건을 사보자." | 장터 |
| 화물이 있음 | "다른 도시에서 팔아보자." | 지도 |
| 100냥 근처 | "조금 더 모으면 손수레야." | 탈것 |
| 300냥 근처 | "집으로 돌아갈 준비가 됐어." | 집으로 |

### C. 장터 화면

| 항목 | 설계 |
|---|---|
| 역할 | 사고파는 재미를 가장 빠르게 느끼는 화면. |
| 도시별 특산품 카드 | `이 도시 물건` 리본을 붙인다. |
| 가격 표현 | 구매는 `산 값`, 판매는 `파는 돈`만 쓴다. |
| 구매 힌트 | "여기서 사기 좋아." |
| 판매 힌트 | "대구에서 인기 많아.", "다른 장터에서 값이 좋아." |
| 구매 후 창 유지 | 기존처럼 장터는 닫히지 않는다. 바로 다음 행동을 고르게 한다. |
| 짐 싣기 애니메이션 | 상품 그림이 짐칸으로 들어가는 짧은 반응을 유지/강화. |
| 돈 변화 피드백 | `-10냥`, `+16냥`이 카드 위 또는 짐칸 옆에 잠깐 뜬다. |

카드 정보 제한:

| 허용 | 금지 |
|---|---|
| 상품 그림, 상품명, 산 값, 파는 돈 | 평균가, 마진, 수익률 |
| 여기서 사기 좋아 | 수요, 공급 |
| 이 도시에서 인기 많아 | 판매처 |
| 짐칸이 찼어 | 적재 효율 |

### D. 지도 화면

| 항목 | 설계 |
|---|---|
| 역할 | 내가 도시를 고르고 떠난다는 감각을 준다. |
| 한국 전도 | 기존 지도 자산을 유지하고 도시 노드를 크게 읽히게 한다. |
| 도시 노드 | 현재 위치, 갈 수 있는 곳, 선택한 곳을 색/크기/파동으로 구분. |
| 육로/해로 구분 | 육로는 따뜻한 선, 해로는 푸른 선. 해로는 배 필요 문구. |
| 선택 도시 카드 | 도시명, 대표 물건 1-2개, 한 문장 소개. |
| 길 카드 | 산길/바닷길/장터길/북방길, 요정 한마디, 출발 버튼. |
| 출발 버튼 | 항상 가장 큰 버튼. 못 가는 길은 이유를 짧게 표시. |
| 현재 위치 강조 | 도시 노드와 상단 상태바에서 모두 보인다. |

길 카드 예시:

| routeType/scenery | Junior 이름 | 문구 |
|---|---|---|
| `mountain` | 산길 | "고개를 천천히 넘어가자." |
| `sea` | 바닷길 | "배가 있으면 갈 수 있어." |
| `marketRoad` | 장터길 | "사람들이 오가는 길이야." |
| `river` | 강가 길 | "물길을 따라 가보자." |
| `north` | 북방길 | "먼 장터로 가는 길이야." |

### E. 이동 화면

| 항목 | 설계 |
|---|---|
| 역할 | 버튼 클릭 사이의 빈 시간을 모험으로 바꾼다. |
| 루트 전용 장면 | `travelSceneAsset`, `scenery`, `routeType`에 맞는 배경 사용. |
| 수레/배 움직임 | 현재 탈것에 맞게 움직임과 소리를 구분. |
| 요정 한마디 | "수레가 고개를 넘어가.", "물결이 반짝여." |
| 사건 가능 | 이동 중 사건이 생길 수 있음을 부드럽게 예고. |
| 도착 예고 | 목적지 이름과 `곧 도착해` 카드. |
| 길이 | 약 5-6초. 아이가 볼 수 있지만 답답하지 않게. |

### F. 사건/퀴즈 화면

| 항목 | 설계 |
|---|---|
| 역할 | 이동 중 긴장과 작은 성취를 만든다. |
| 사건 그림 | 사건 종류를 상징하는 큰 그림. 무섭게 묘사하지 않는다. |
| 짧은 상황 | 제목 1줄, 요정 문구 1줄. |
| 퀴즈 종류 | 맞춤법, 숫자, 선택 퀴즈. 초반은 2지선다. |
| 정답 결과 | "잘 골랐어!", `별 +1`, `돈 +2냥` 등. |
| 오답 결과 | "괜찮아, 천천히 가자.", 작은 손실 또는 보상 없음. |
| 변화 표시 | 돈/별/짐 변화는 결과 카드에서 칩으로 표시. |

오답 처리 원칙:

| 상황 | 처리 |
|---|---|
| 첫 오답 | 보상 없음 또는 `돈 -1냥` 수준 |
| 연속 오답 | 요정 힌트 추가 |
| 짐 손실 | 아주 드물게 `짐 하나 잃음`, 즉시 "계속 갈 수 있어" 표시 |
| 게임오버 | 사용하지 않음 |

### G. 탈것 장만 화면

| 항목 | 설계 |
|---|---|
| 역할 | 돈을 모으는 이유를 보여준다. |
| 현재 탈것 | 현재 수레/배와 짐칸 수를 크게 표시. |
| 다음 탈것 | 살 수 있는 다음 목표를 먼저 보여준다. |
| 필요한 돈 | "손수레까지 40냥 남았어." |
| 짐칸 증가 | `짐칸 +1`처럼 짧게 표시. |
| 새 길 열림 | "작은 나룻배가 있으면 바닷길을 건너." |
| 구매 축하 | 구매 즉시 축하 카드와 짧은 소리/움직임. |

탈것 목표:

| 탈것 | 아이에게 보이는 효과 |
|---|---|
| 보따리 수레 | 짐칸 2칸 |
| 손수레 | 짐을 하나 더 실어 |
| 큰 수레 | 먼 장터도 든든해 |
| 장사 수레 | 큰 장사 준비 끝 |
| 작은 나룻배 | 바닷길을 건널 수 있어 |
| 작은 돛배 | 제주 가기 쉬워 |

### H. 목표/도장 화면

| 항목 | 설계 |
|---|---|
| 역할 | 장기 목표를 아이가 볼 수 있는 수집판으로 만든다. |
| 도시 도장 | `visitedCityIds`를 도장판으로 표시. |
| 배지 | `꼬마 장사꾼`, `도시 도장 3개`, `제주까지 다녀왔어요` 등. |
| 엔딩 목표 | `300냥까지 n냥`으로 표시. |
| 아직 못 가본 곳 | 지도 힌트로 "아직 못 가본 도시가 있어." |
| UI 밀도 | 한 화면에 도장 6-8개 정도만 크게, 나머지는 스크롤. |

이 화면은 처음부터 별도 복잡한 메뉴로 만들지 않아도 된다. Milestone 1에서는 도시 화면의 목표 카드와 진행 카드로 시작하고, 후속에서 별도 도장 화면으로 확장한다.

### I. 엔딩 화면

| 항목 | 설계 |
|---|---|
| 역할 | 300냥 목표 달성의 완결감을 준다. |
| 장부가 빛남 | 정우가 장부를 들고 돌아가는 장면. |
| 집으로 돌아가기 | `집으로 가기`를 기본 버튼으로 둔다. |
| 계속 장사하기 | 엔딩 이후 자유 플레이 가능. |
| 다시 하기 | 저장 초기화 전 확인은 후속 후보. |
| 결과 | `꼬마 거상 정우` 배지, 모은 도장/별 요약. |

## 3. 데이터 설계

현재 코드의 타입과 맞추되, 후속 확장 때의 목표 필드를 아래처럼 정리한다. 첫 구현은 새 필드를 모두 추가하지 않고 기존 데이터에서 파생하는 것을 우선한다.

### JuniorCity

| 필드 | 의미 | 현재 대응 |
|---|---|---|
| `id` | 도시 ID | 기존 `JuniorCity.id` |
| `name` | 도시명 | 기존 |
| `region` | 권역명 | 기존 |
| `cityType` | 내륙/항구/섬/북방 등 | 현재 `kind`와 대응 |
| `backgroundAsset` | 도시 배경 | 기존 `backgroundAsset`, `scene` |
| `mapPosition` | 지도 좌표 | 현재 `x`, `y` |
| `buyGoodIds` | 이 도시에서 사기 좋은 물건 | 기존 |
| `sellGoodIds` | 이 도시에서 인기 많은 물건 | 기존 |
| `connectedCityIds` | 연결 도시 | 현재 `JUNIOR_ROUTES`에서 파생 |
| `routeTypes` | 이 도시 주변 길 성격 | 현재 route의 `routeType`에서 파생 |
| `introDialogues` | 첫 방문 소개 | 현재 `introLines` |
| `stampLabel` | 도장 문구 | 신규 후보. 없으면 도시명 사용 |

### JuniorGood

| 필드 | 의미 | 현재 대응 |
|---|---|---|
| `id` | 상품 ID | 기존 |
| `name` | 상품명 | 기존 |
| `icon` | 상품 그림 | 현재 `image` |
| `baseBuyPrice` | 기본 산 값 | 현재 `baseBuyCoins` |
| `baseSellPrice` | 기본 파는 돈 | 현재 `baseSellCoins` |
| `originCityIds` | 이 도시 물건 | 현재 도시 `buyGoodIds`에서 역산 |
| `popularCityIds` | 인기 많은 도시 | 현재 도시 `sellGoodIds`에서 역산 |
| `childHint` | 자연어 힌트 | 신규 후보. 예: "가볍고 팔기 좋아." |

### JuniorRoute

| 필드 | 의미 | 현재 대응 |
|---|---|---|
| `from` | 출발 도시 | 기존 |
| `to` | 도착 도시 | 기존 |
| `routeType` | 산길/바닷길/장터길 등 | 기존 `routeType`, `scenery`, `kind` |
| `sceneAsset` | 이동 배경 | 현재 `travelSceneAsset` |
| `eventDeckIds` | 사건 후보 | 현재 `eventCategories`, `routeTypes` |
| `requiresBoat` | 배 필요 | 현재 `needsBoat` |
| `requiresVehicle` | 수레 필요 | 신규 후보. 초반에는 쓰지 않음 |
| `fairyHint` | 길 카드 문구 | 현재 `fairyText`, `arrivalHint` |

### JuniorEvent

| 필드 | 의미 | 현재 대응 |
|---|---|---|
| `id` | 사건 ID | 기존 |
| `category` | 사건 종류 | 현재 `type`, `mood` |
| `title` | 제목 | 기존 |
| `sceneAsset` | 사건 그림 | 현재 `scene`과 `eventSceneImages` |
| `fairyText` | 요정 문구 | 기존 |
| `quiz` | 맞춤법/숫자/선택 퀴즈 | 기존 |
| `choices` | 선택지 | 기존 |
| `reward` | 정답/성공 보상 | 기존 |
| `penalty` | 오답/작은 손실 | 현재 `wrongReward`, `loseCargo`와 대응 |
| `storyArcId` | 이야기 흐름 | 기존 |
| `once` | 한 번만 보기 | 신규 후보. 현재 `seenEventIds`로 처리 가능 |

### JuniorState

| 필드 | 의미 | 현재 대응 |
|---|---|---|
| `currentCityId` | 현재 도시 | 기존 |
| `coins` | 돈 | 기존 |
| `stars` | 별 | 기존 |
| `cargo` | 짐 | 기존 |
| `cargoLimit` | 짐칸 | 기존 |
| `vehicleId` | 현재 수레 | 기존 |
| `boatId` | 현재 배 | 기존 |
| `visitedCityIds` | 방문 도시 | 기존 |
| `cityStamps` | 도시 도장 | 신규 필드보다 `visitedCityIds` 파생 우선 |
| `badges` | 배지 | 기존 |
| `storyArcProgress` | 이야기 진행 | 기존 |
| `completedTutorial` | 튜토리얼 완료 | 기존 |
| `completedEnding` | 엔딩 완료 | 기존 |
| `lastSavedAt` | 저장 시각 | 기존 |

## 4. 튜토리얼 재설계

튜토리얼은 full mode의 첫 30분 구조를 junior용 5-7분 구조로 줄인다. 목표는 설명을 듣는 것이 아니라 직접 한 번 장사를 성공하는 것이다.

| 단계 | 한 문장 대사 | 강조 버튼 | 완료 조건 | 다음 단계 |
|---|---|---|---|---|
| 1. 조선에 도착 | "정우야, 장사를 시작하자." | 시작하기 | `startIntro` 실행 | 부산 도시 |
| 2. 부산 장터 보기 | "부산 장터를 둘러보자." | 장터 | `currentStep='market'` | 상품 보기 |
| 3. 특산품 하나 사기 | "면포를 하나 골라보자." | 면포 카드 | `selectedGoodId` 또는 구매 클릭 | 짐 싣기 |
| 4. 짐에 싣기 | "짐에 실렸어." | 짐에 싣기 | `cargo.length >= 1` | 지도 |
| 5. 지도에서 대구 선택 | "대구로 가보자." | 대구 | `destinationCityId='daegu'` 후보 | 길 카드 |
| 6. 길 카드 확인 | "산길이야. 천천히 가자." | 출발 | `currentStep='travel'` | 이동 |
| 7. 이동 | "수레가 고개를 넘어가." | 도착하기 | `finishTravel` 실행 | 사건 |
| 8. 작은 사건/퀴즈 | "바른 말을 골라보자." | 정답 보기 | `currentStep='eventResult'` | 도착 |
| 9. 대구 도착 | "대구에 도착했어." | 둘러보기 | `currentStep='city'` | 장터 |
| 10. 팔기 | "면포를 팔아보자." | 팔기 | `cargo.length` 감소, `coins` 증가 | 보상 |
| 11. 돈/별 획득 | "잘했어! 돈이 늘었어." | 계속하기 | `stars` 또는 `message` 표시 | 성장 목표 |
| 12. 손수레 목표 안내 | "조금 더 모으면 손수레야." | 탈것 보기 | 목표 카드 표시 | 자유 루프 |

튜토리얼의 실패 허용:

| 문제 | 처리 |
|---|---|
| 다른 상품을 삼 | 그 상품의 좋은 판매 도시를 안내한다. |
| 대구 대신 다른 도시 선택 | 초반에는 대구를 더 강하게 추천하되 막지는 않는다. |
| 퀴즈 오답 | 작은 손실 또는 보상 없음. 같은 흐름 계속. |
| 돈이 부족함 | "한 개만 사도 괜찮아." 안내. |

## 6. UX 기준

| 항목 | 기준 |
|---|---|
| 첫 10초 | 할 일과 누를 버튼이 보여야 한다. |
| 첫 성공 | 3-5분 안에 구매, 이동, 판매 성공. |
| 첫 성장 목표 | 5-8분 안에 손수레 목표 노출. |
| 반복 목표 | 10분 내 손수레, 15-20분 작은 배, 20-30분 300냥 엔딩. |
| 버튼 | 주요 버튼 56px 이상. |
| 카드 밀도 | 초반 화면은 3개 이하 선택을 우선. |
| 모바일 | 390x844, 412x915, 430x932 세로 우선, 844x390 가로도 깨지지 않게. |
| 저장 | 모든 주요 행동 후 자동 저장. |

## 7. 제외 기준

| 제외 | 이유 |
|---|---|
| full mode 장부/허가/평판 구조 | junior 목표를 흐린다. |
