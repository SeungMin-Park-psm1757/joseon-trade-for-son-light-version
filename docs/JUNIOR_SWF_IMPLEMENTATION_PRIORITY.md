# Junior SWF Implementation Priority

Last updated: 2026-05-14

목적: full mode SWF-inspired loop를 junior light mode에 적용할 때의 구현 순서를 정한다. 이 문서는 구현 전 우선순위 문서이며, 아직 `junior-game/` 코드를 수정하지 않는다.

## A. 즉시 구현해야 할 것

| 우선순위 | 항목 | 목표 | 예상 파일 | 완료 기준 |
|---:|---|---|---|---|
| 1 | 오늘 할 일 카드 | 첫 10초 안에 다음 행동을 알게 한다. | `junior-game/src/JuniorApp.tsx`, `juniorStyles.css`, `juniorFlow.ts` 후보 | 모든 주요 화면에 한 문장 목표와 핵심 버튼이 보인다. |
| 2 | 시장 가격 표현 정리 | full mode 가격 정보를 junior 자연어로 바꾼다. | `JuniorApp.tsx`, `juniorStyles.css` | 상품 카드가 `산 값`, `파는 돈`, `여기서 사기 좋아`, `여기서 인기`만 보여준다. |
| 3 | 길 카드 | 출발 전 이동이 기대되는 선택이 되게 한다. | `JuniorApp.tsx`, `juniorStyles.css`, 기존 `JUNIOR_ROUTES` | 도시 선택 시 산길/바닷길/장터길과 요정 한마디, 출발 버튼이 보인다. |
| 4 | 사건 결과 보상 카드 | SWF식 짧은 성공/실패 리듬을 만든다. | `JuniorApp.tsx`, `juniorStyles.css` | 사건 뒤 `+냥`, `별 +1`, `괜찮아`가 1초 안에 읽힌다. |
| 5 | 수레/배 목표 카드 | 첫 장사 뒤 성장 목표가 보이게 한다. | `JuniorApp.tsx`, `juniorStyles.css` | 손수레/작은 나룻배까지 남은 돈과 효과가 짧게 보인다. |
| 6 | 도시 도장 강화 | full mode 장부 목표를 아이용 수집 목표로 바꾼다. | `JuniorApp.tsx`, `juniorStyles.css` | 방문 도시 수와 다음 도장 목표가 보인다. |
| 7 | 장터 구매/판매 피드백 | 구매/판매가 즉시 즐겁게 느껴지게 한다. | `JuniorApp.tsx`, `juniorStyles.css`, 기존 audio | 구매 후 "대구에서 인기 많아", 판매 후 `+냥`이 보인다. |
| 8 | 엔딩 목표 힌트 | 300냥 목표가 너무 멀게 느껴지지 않게 한다. | `JuniorApp.tsx`, `juniorStyles.css` | "300냥까지 n냥"과 "집으로 돌아가기" 목표가 보인다. |
| 9 | QA 안정 식별자 | UI 변경 후 smoke test가 흔들리지 않게 한다. | `junior-game/tests/junior-smoke.spec.ts`, UI 요소 | 신규 주요 카드에 안정적인 `data-testid`가 있다. |
| 10 | 저장 호환 확인 | 기존 저장을 깨지 않는다. | `juniorFlow.ts`, tests | 이전 저장값이 정상 normalize되고 새 UI 파생 목표가 저장 필수값을 요구하지 않는다. |

## B. 다음 구현

| 항목 | 목표 | 이유 |
|---|---|---|
| 길 타입별 사건 카드 톤 정리 | 산길, 바닷길, 장터길, 북방길의 느낌을 다르게 한다. | 이동 반복이 덜 지루해진다. |
| 오늘의 추천 상품 | full mode 월별 유행품을 아이용 "오늘 추천"으로 변환한다. | 경제 변화 감각을 숫자 없이 줄 수 있다. |
| 작은 배 목표 강화 | 제주/바닷길을 장기 목표로 부드럽게 예고한다. | 15-20분 목표가 선명해진다. |
| 도장판 화면 후보 | 방문 도시와 배지를 더 크게 보여준다. | 수집 동기를 강화한다. |
| 퀴즈 난이도 자동 완화 | 오답 연속 시 더 쉬운 보기와 힌트를 준다. | 저학년 좌절을 줄인다. |
| 첫 5분 튜토리얼 카피 정리 | 한 컷 한 문장 원칙을 적용한다. | 보호자 설명 없이 시작하기 쉬워진다. |
| 모바일 landscape 보정 | 844x390에서도 핵심 카드가 잘리지 않게 한다. | 기존 방향 업데이트와 맞춘다. |
| 사운드/모션 피드백 정돈 | 구매, 판매, 정답, 장비 구매의 느낌을 구분한다. | SWF식 즉시 반응을 어린이용으로 살린다. |

## C. 후속 후보

| 추천 루트 2개 비교 | 초2 이상 playtest에서 선택 여유가 확인될 때만 추가한다. |
| 계절/날씨 말풍선 | 경제 설명 없이 분위기와 추천만 줄 수 있을 때 추가한다. |

## D. 제외할 것

| 제외 항목 | 제외 이유 |
|---|---|
| 평균가 | junior 시장을 표처럼 만든다. |
| 수익률, 마진, 시세차익 | 초1-초2에게 단어와 개념이 어렵다. |
| 장부 조각 복잡도 | 도시 도장과 배지로 충분하다. |
| 상세 평판, 항구 신뢰, 호위 명성 | 보상 축이 너무 많아진다. |
| 다중 장비 수치 | `짐칸 +1`, `바닷길 좋아` 이상의 수치는 숨긴다. |
| 복잡한 전투 | junior는 전략 전투보다 작은 사건/퀴즈가 맞다. |
| 수요/공급 설명 | `여기서 인기`, `이 도시 물건`으로 충분하다. |
| 위험도 숫자와 성공률 | 길 카드가 계산표가 된다. |
| 내구도, 수리비, 보급품 | 실패감을 키우고 장비 UI를 복잡하게 한다. |
| SWF 원본 자산/코드/사운드/문구 | 저작권상 금지이며 분석 목적과 다르다. |

## 2. 권장 구현 순서

| Milestone | 범위 | 테스트 |
|---|---|---|
| Junior A: 첫 5분 안내 | 오늘 할 일 카드, 첫 시장/지도/판매 목표 문구 | `cd junior-game && npm run build && npm run test` |
| Junior B: 길 카드와 사건 결과 | 지도 선택 카드, 결과 보상 칩, 오답 완충 문구 | 튜토리얼 smoke, 이벤트 quiz smoke |
| Junior C: 성장 목표 | 손수레/배 목표 카드, 도장/엔딩 목표 힌트 | 저장/이어하기, 상점 구매, 엔딩 선택 |
| Junior D: playtest 조정 | 글량, 버튼 크기, 가격/보상, 퀴즈 난이도 | 390x844, 412x915, 430x932, 844x390 수동 확인 |

## 3. 구현 전 체크

1. 현재 작업트리에 남은 기존 `junior-game/` 변경을 별도 처리한다.
2. `starter/` full mode 변경과 junior 변경을 한 커밋에 섞지 않는다.
3. `reference-swf/original/`, `extracted/`, `screenshots/`, `tools-notes/`는 게임 번들에 포함하지 않는다.
4. 새 데이터 필드는 첫 구현에서 최소화하고, 가능한 파생 UI로 시작한다.
5. `joseon_trade_junior_save_v1` 저장 호환을 유지한다.
6. 새 UI 요소에는 smoke test용 안정 `data-testid`를 붙인다.

## 4. 다음 구현 프롬프트 초안

```text
너는 junior-game의 principal React/TypeScript child-game implementer다.

목표:
docs/JUNIOR_SWF_TRANSLATION_TABLE.md,
docs/JUNIOR_SWF_INSPIRED_CORE_LOOP.md,
docs/JUNIOR_SWF_IMPLEMENTATION_PRIORITY.md의 Milestone Junior A만 구현하라.

중요:
- full mode 기능을 그대로 이식하지 않는다.
- `starter/`는 건드리지 않는다.
- SWF 자산, 코드, 사운드, 문구는 사용하지 않는다.
- 기존 `joseon_trade_junior_save_v1` 저장 호환을 유지한다.
- 현재 작업트리의 기존 junior 변경 범위를 먼저 확인하고, 새 변경과 섞이지 않게 보고한다.

필수 구현:
1. `오늘 할 일` 카드 추가
2. 시장 상품 카드 문구를 `산 값`, `파는 돈`, `여기서 사기 좋아`, `여기서 인기` 중심으로 정리
3. 부산 -> 대구 첫 장사 흐름에서 다음 버튼이 분명하게 보이도록 개선
4. 구매/판매 결과 피드백을 한 문장과 `+냥` 중심으로 정리
5. 주요 신규 UI에 안정 `data-testid` 추가 또는 기존 testid 유지

검증:

npm run test

수동 확인:
새 게임 -> 부산 장터 -> 면포 구매 -> 지도 -> 대구 이동 -> 퀴즈 -> 대구 도착 -> 판매 -> 손수레 목표 확인
```

## 5. 진행 판정

판정: 설계상 진행 가능, 구현은 작업트리 정리 후 시작.

이유:

- full mode SWF-inspired 요소는 junior용 언어로 번역되었다.
- 가져올 것과 제외할 것이 분리되었다.
- 현재 `junior-game`에는 이미 필요한 화면과 저장 구조가 있어 첫 구현은 파생 UI 중심으로 가능하다.
- 단, gate report 기준으로 기존 `junior-game/` 잔여 변경이 많으므로 구현 브랜치/커밋 범위를 먼저 정리해야 한다.
