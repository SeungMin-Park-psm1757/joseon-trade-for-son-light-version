# MVP 범위 정의

## 1. MVP 핵심 문장

모바일 화면에서 플레이어가 조선 후기 상인이 되어 한국 남부·제주·대마도 지도를 이동하고, 상품을 사고팔며, 월별 가격 변화와 위험 이벤트를 겪고, 돈으로 배와 수레를 사는 1차 프로토타입.

## 2. 반드시 동작해야 하는 루프

```text
새 게임 시작
→ 부산포 도착
→ 시장에서 상품 구매
→ 지도에서 이동지 선택
→ 이동 중 이벤트 발생 가능
→ 도착 항구에서 판매
→ 이익 확인
→ 월이 바뀌면 가격 갱신
→ 돈을 모아 배/수레 구매
→ 제주 또는 대마도 목표 도전
```

## 3. MVP 포함 기능

| 기능 | MVP 기준 |
|---|---|
| 새 게임/이어하기 | LocalStorage 기반 |
| 지도 | 추상화된 2D 지도, 항구 아이콘 클릭 |
| 항구 | 18개 거점 데이터 사용 |
| 상품 | 28개 상품 데이터 사용 |
| 이동 | routes.json 연결 기반 |
| 경제 | 월별 가격표 생성 |
| 시장 | 구매/판매/화물칸 제한 |
| 선박 | 구매, 현재 배 능력치 반영 |
| 수레 | 구매, 육상 화물칸 반영 |
| 어업 | 항구에서 하루 소모 후 보조 수입 |
| 이벤트 | 위험 이벤트 7개 이상 구현 |
| 전투 | 자동 또는 4선택지 간단 전투 |
| 퀘스트 | 튜토리얼 의뢰 3개 이상 |
| 저장 | 거래·이동·이벤트 후 자동 저장 |

## 4. MVP 제외 기능

| 제외 기능 | 이유 |
|---|---|
| 중국 본토 | 1차 지도 범위 과대화 방지 |
| 일본 본토 | 대마도까지만 목표화 |
| 항구 투자 | 경제 루프 검증 후 추가 |
| 실시간 해전 | 구현 부담 대비 효과 낮음 |
| 멀티플레이 | 가족용 싱글 우선 |
| 서버 저장 | 모바일 웹 MVP는 LocalStorage로 충분 |
| 정교한 조석표 | 간단한 low/middle/high 주기로 대체 |
| 에셋 다운로드 자동화 | 라이선스 검수 필요 |

## 5. 30분 플레이 목표

- 5분: 시작, 지도, 시장 이해
- 10분: 첫 거래 성공
- 20분: 위험 이벤트 1회 경험
- 30분: 손수레 또는 어선 구매 목표가 보임

## 6. 1시간 플레이 목표

- 남해 또는 서해 주요 항로 경험
- 제주 항로 도전 준비
- 최소 3개 퀘스트 완료
- 월별 가격 변동 1회 이상 경험
- 새 운송수단 구매

## 7. MVP 성공 기준

- 아이가 “어디서 사서 어디서 팔지”를 스스로 고를 수 있다.
- 돈을 벌고 커지는 감각이 있다.
- 위험 이벤트가 성가신 장식이 아니라 선택을 요구한다.
- 모바일 화면에서 손가락으로 조작 가능하다.
- 데이터 JSON을 수정하면 게임 내용이 바뀐다.
