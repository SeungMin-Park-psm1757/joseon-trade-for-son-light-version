# 팔도상단: 조선의 바람 - 현재 게임 구성 상세 문서

Last updated: 2026-05-13

이 문서는 ChatGPT나 새 작업자가 현재 게임의 구조와 의도를 빠르게 이해하도록 정리한 기준 문서다. 실제 구현의 중심은 `starter/src/App.tsx`, 타입 정의는 `starter/src/types.ts`, 오디오 시스템은 `starter/src/audio.ts`, 데이터는 `starter/public/data/*.json`에 있다.

## 1. 게임 한 줄 요약

`팔도상단: 조선의 바람`은 조선 후기의 항구, 장터, 산길, 해로를 배경으로 하는 모바일 웹 무역 RPG다. 현재 저장소에는 두 게임 경험이 공존한다. Full Mode는 `정우의 거상 모험`이고, Junior Light Mode는 `정우의 첫 장사놀이`다.

## 2. 핵심 방향

- 플랫폼: 모바일 웹, React + Vite + TypeScript.
- 저장: 서버 없이 localStorage 기반.
- 기준 화면: 가로형 `844x390`, `932x430`, 데스크톱 `1365x768`, `1920x1080`.
- 세로 화면: 억지로 모든 UI를 맞추지 않고 가로 플레이 안내를 우선한다.
- 표현 방향: 텍스트 앱이 아니라 픽셀아트 장면, 지도, 핫스팟, 장터, NPC, 사운드 중심의 조선 무역 RPG.
- 서사 방향: 정우가 낯선 조선에서 첫 면포 장사를 배우고, 돈을 모아 수레와 배를 마련해 조선의 거상으로 성장하는 어린이 친화형 시간여행 이야기.
- 금지 범위: 중국 본토 확장, 일본 본토 확장, 항구 투자, 실시간 해전, 서버 저장, 로그인, Phaser/PixiJS 전환.

## 2.1. 초반 서사와 튜토리얼

- 주인공: 정우.
- 시작 상황: 정우가 현대에서 갑자기 조선 부산포로 오고, 시간여행 요정이 장사길을 안내한다.
- 튜토리얼 표현: 하단 공지 위젯이 아니라 `tutorial-dialog` 모달 대화창으로 표시된다. 대화가 뜨면 화면이 어두워지고 게임 조작은 일시 정지된다.
- 첫 대화 핵심: 부산포에 도착한 정우를 안심시키고, 작은 면포 장사부터 큰 상단까지의 장기 목표를 제시한다.
- 진행 안내: 대화 모달은 큰 전환점에서만 뜨며, 평소에는 작은 다음 행동 스트립과 첫 30분 성장 루트가 시장/지도/장비로 바로 이어진다.
- 어린이용 용어: `매수가/매도가/적재/화물칸/예상 수익/항로` 같은 어려운 표현은 화면 주요 문구에서 `사는 값/파는 값/짐칸/벌 돈/가는 길/바닷길`처럼 바꾸었다.

## 3. 현재 플레이 루프

```text
새 게임 시작
→ 부산포 항구 장면 입장
→ 시장/관청/술집/조선소/어업/출항 핫스팟 선택
→ 시장에서 상품 아이콘을 눌러 구매
→ 지도에서 목적지와 루트 선택
→ 배 또는 수레가 이동하는 연출
→ 위험 이벤트 또는 이동 결과 확인
→ 도착 항구에서 판매 또는 의뢰 진행
→ 거래 장부, 명성, 항구 신뢰, 발견 도감 갱신
→ 돈을 모아 수레/배/개인 장비/동료를 강화
→ 더 먼 장사길, 제주, 대마도, 북방 루트 준비
```

초반 목표는 아이가 그림과 버튼만 보고도 따라갈 수 있는 첫 10~30분 루프다.

1. 부산포 시장에서 면포 구매.
2. 대구로 이동.
3. 대구 시장에서 면포 판매.
4. 소금 장사 시작.
5. 손수레 구매.
6. 어업 시도.
7. 어선 준비.
8. 남해 항로 확장.
9. 제주 준비.
10. 왜관 허가장과 대마도 교역 준비.

## 3.2 2026-05-13 Full Mode SWF 분석 반영 패스

SWF reference 분석에서 추출한 "짧은 선택 → 이동 긴장감 → 결과 칩 → 다음 목표" 리듬을 조선 후기 무역 RPG 구조로 변환했다. 자산, 코드, 문구는 복사하지 않았고, full mode의 데이터와 UI만 확장했다.

- 모든 주요 화면에 현재 목표와 다음 추천 행동을 보여주는 full mode 목표 스트립을 추가했다.
- 시장은 산 값/파는 값/평균가/추천 판매처/예상 이익/가격 이유를 함께 보여준다.
- 같은 항구 즉시 되팔기는 손해가 보이도록 표시하고, 인접 루트 추천을 통해 이동 목적을 만든다.
- 지도는 루트별 위험 프로필, 준비물, 계절 위험, 추천 상품, 보상 힌트를 표시한다.
- 첫 이동에는 도적/길손 상인/표류선 구조 같은 사건이 반드시 체감되도록 사건 덱 우선순위를 보정했다.
- 사건 모달은 대응 준비물, 결과 칩, 선택 결과 힌트, 다음 추천 행동을 짧게 보여준다.
- 배와 수레는 가격과 해금 힌트를 재정리해 손수레, 어선, 연안 상선으로 이어지는 목표성이 강해졌다.
- 장부 화면은 최근 거래와 장부 조각 진행뿐 아니라 다음 발견/다음 장사길 안내를 더 분명하게 보여준다.

## 3.1 Junior Light Mode

Junior Light Mode는 초등학교 1학년용 별도 게임이다. 기존 full mode의 UI를 축소하지 않고 `junior-game/` 아래에서 독립 실행되는 세로형 모바일 게임으로 렌더링한다. `starter/`의 full mode 컴포넌트, CSS, 저장 키를 직접 사용하지 않는다.

모드 이름:

- Full Mode: 정우의 거상 모험
- Junior Light Mode: 정우의 첫 장사놀이

Junior mode의 핵심 루프:

```text
도입
→ 큰 상품 그림 3개 중 하나 고르기
→ 짐에 싣기
→ 부산포에서 대구로 가는 짧은 이동 연출
→ 팔기
→ 돈 증가
→ 첫 장사 성공 배지
→ 한 번 더 하기 또는 큰 모험 보기
```

Junior mode에서는 가격 비교표, 위험도, 의뢰, 명성, 장비, 장부, 배 정박지, 허가장, 이벤트를 화면에 보여주지 않는다. 요정이 큰 초상과 말풍선으로 정우를 부르며 안내하고, 한 화면에는 한 가지 행동만 둔다.

## 6. 항구와 도시

현재 거점은 조선 팔도 남부, 동해, 서해, 제주, 대마도, 북방까지 포함한다.

- 중앙/내륙: 한양, 충주, 전주, 대구, 안동, 개성
- 서해/서남해: 강화, 군산포, 목포, 흑산도, 남포
- 남해: 여수, 통영, 제포, 부산포
- 동해: 울산, 영일만, 강릉, 원산, 함흥
- 섬/외항: 제주, 대마도
- 북방/국경: 평양, 남포, 의주, 경흥, 책문 장시, 두만강 북방장

UI 표시에서는 조선 후기 세계관을 우선해 한양, 부산포, 의주, 경흥처럼 시대감 있는 이름을 쓴다. 단, 저장 호환성을 위해 일부 데이터 ID는 기존 이름을 유지한다.

항구별 개성은 `port_flavors.json`에서 관리한다.

- 대표 상품
- 항구 설명
- 시장/관청/술집/조선소 NPC 대사
- 핫스팟 좌표
- 시장 진열 좌표
- 지역 소문
- 추천 행동

## 7. 화면 구조

탭은 다음 7개다.

- `port`: 항구 메인
- `market`: 시장
- `map`: 지도/이동
- `cargo`: 화물
- `quests`: 의뢰
- `vehicles`: 장비/동료/배/수레
- `ledger`: 장부/도감

### 7.1 공통 HUD

상단에는 항상 핵심 상태가 보인다.

- 현재 위치
- 보유금
- 날짜/월
- 월간 사건
- 화물 상태
- 사운드 버튼
- 자동 저장 상태

UI 톤은 한지색 바탕, 먹색/남색 반투명 패널, 청록/황동 CTA, 픽셀아트 이미지 중심이다.

### 7.2 항구 메인

항구 메인은 카드형 대시보드가 아니라 항구 배경 장면 위의 핫스팟 구조다.

주요 핫스팟:

- 시장
- 관청
- 술집
- 조선소/수레방
- 어업
- 출항/지도

핫스팟은 지역 배경 위에 배치된다. 시장은 마을/장터 쪽, 관청은 언덕이나 관아 쪽, 조선소는 물가나 선착장 쪽에 놓는 방향이다.

오른쪽/측면에는 오늘의 항구 정보가 짧게 표시된다.

- 항구명
- 특산품
- 이번 달 소식
- 추천 행동
- 현재 화물 요약
- 배 정박 상태

### 7.3 시장

시장은 표가 아니라 장터 진열대다.

동작:

1. 항구별 판매 상품 3~8개가 상품 아이콘으로 배치된다.
2. 상품 아이콘을 누르면 거래 팝업이 열린다.
3. `사기` 버튼은 1개씩 구매한다.
4. `팔기` 버튼은 1개씩 판매한다.
5. 거래 후 `+면포`, `-25냥` 같은 플로팅 피드백이 뜬다.
6. 돈 부족, 화물칸 부족, 보유량 부족은 짧은 상태칩으로 표시한다.

거래 팝업 정보:

- 상품 아이콘
- 상품명
- 현재 구매가
- 현재 판매가
- 보유 수량
- 평균 매입가
- 지금 팔면 손익
- 추천 판매처
- 사기/팔기 큰 버튼

시장 추천은 인접 루트 기준으로 계산한다.

- 지금 팔 곳: 보유 화물을 가까운 항구에서 더 비싸게 팔 수 있는지 계산.
- 다음에 살 것: 현재 항구에서 사서 가까운 항구에 팔 때 예상 이익이 큰 상품 계산.

### 7.4 지도/이동

지도는 한반도 전도 기반의 픽셀아트 게임 지도다. 추상 폴리곤 섬이 아니라 서해, 남해, 동해, 제주, 대마도 위치가 드러나는 구조를 목표로 한다.

지도 화면 구성:

- 왼쪽: 한반도 지도, 도시 노드, 육로/해로, 현재 위치, 배 정박 위치, 추천 루트, 이동 애니메이션.
- 오른쪽: 선택한 목적지, 이동 수단, 소요일, 위험도, 적재 가능 여부, 예상 이익, 출발 버튼.

루트는 `routes.json`에 있고 `mode`로 구분한다.

- `land`: 수레/지게/마차 이동.
- `sea`: 배 이동.

중요 규칙:

- 배는 장비가 아니라 정박지에 머무르는 이동 수단이다.
- `shipPortId`가 현재 배의 정박 위치다.
- 육로 사건은 선박을 손상시키지 않는다.
- 해로 사건은 배를 손상시킬 수 있다.
- 배가 다른 항구에 있으면 사공/NPC를 통해 호출해야 한다.
- 내륙 도시에서는 배를 직접 호출할 수 없고 가까운 항구로 이동해야 한다.

### 7.5 의뢰

의뢰 화면은 텍스트 목록이 아니라 첫 30분 성장 루트와 현재 목표를 보여주는 화면이다.

현재 의뢰 12개:

- 첫 장사
- 소금은 산길을 넘고
- 건어물 장사
- 손수레 장만
- 남해 항로 개척
- 응급 수리 훈련
- 제주로 가는 첫 짐
- 제주의 말
- 왜관 허가장
- 대마도 첫 교역
- 해적 경계

## 8. 명성, 신뢰, 발견

### 8.1 3대 명성

현재 명성은 3축이다.

- 상인 신용: 교역, 판매 이익, 납품 의뢰, 시장 활동.
- 탐방 명성: 새 항구 방문, 발견 카드, 도감 등록.
- 호위 명성: 도적/해적/위험 사건 해결, 수리, 호송, 위험 루트 생존.

UI에는 긴 설명 대신 칩으로 표시한다.

```text
상인 신용 12
탐방 명성 6
호위 명성 3
```

### 8.2 항구 신뢰도

`portTrust`는 항구별 신뢰도다. 대항해시대식 투자/점유율이 아니라 조선식 관계도다.

오르는 조건:

- 항구 첫 방문 발견
- 해당 항구에서 이익 판매
- 납품 의뢰 완료
- 관련 위험 사건 해결

효과:

- 술집 소문 추가
- 추천 판매처 강조
- 특산품 진열 우선도
- 의뢰 힌트 표시

### 8.3 발견 카드

`discoveries.json`에는 25개 발견 카드가 있다. 항구를 처음 방문하면 자동 등록된다.

예:

- 부산포: 왜관으로 이어지는 장사길
- 통영: 수군 항로
- 여수: 남해 어장
- 목포: 서남해 조류
- 흑산도: 암초 해역
- 강화: 갯벌 나루
- 한양: 관청 장부
- 대구: 약령시
- 제주: 말 목장
- 대마도: 외국 상인 나루
- 개성: 상업 관문

발견 보상:

- 탐방 명성
- 상인 신용 또는 호위 명성 일부
- 돈
- 항구 신뢰도

## 18. 현재 강점

- 교역/이동/판매/저장 루프가 작동한다.
- 의뢰 완료 판정이 상태 기반이다.
- 시장 추천과 지도 추천이 연결된다.
- 배 정박지와 육로/해로 피해 대상이 분리되어 있다.
- 픽셀아트 장면, 핫스팟, 지도, NPC, 오디오가 들어갔다.
- 동료/가족/개인 장비가 성장 루프와 이벤트 조언에 연결된다.
- 발견 카드, 항구 신뢰도, 장부 조각으로 장기 목표가 생겼다.
- BGM/SFX가 장면별로 나뉘어 있다.
- 2026-05-03 기준 세계관은 조선 후기로 고정되었고, 청/러시아 방면은 본토 확장이 아닌 의주·경흥의 제한 국경 장시로 정리되었다.

## 19. 현재 한계

- 고품질 캐릭터 반신 픽셀아트는 아직 부족하다. 일부는 경량 SVG/placeholder다.
- 항구별 핫스팟 좌표는 더 세밀한 개별 조정이 필요하다.
- 시장 진열은 항구별 개성을 더 강화할 수 있다.
- 이벤트 장면은 시스템은 있으나 이미지 품질을 더 올릴 여지가 있다.
- 동료 대사와 이벤트 결과 문구는 일부가 아직 코드 helper에 남아 있다.
- 첫 30분 이후 장기 밸런스는 추가 플레이테스트가 필요하다.
- 북방 거점은 의주/경흥/책문 장시/두만강 북방장으로 정리되었지만, 전용 의뢰와 장면 에셋은 아직 더 깊게 만들 여지가 있다.
- 일본/중국 본토 확장은 금지 범위이며, 현재는 대마도와 언어 준비만 있다.

## 19.1 세계관/북방 교역 기준

- 최종 시대 설정은 19세기 조선 후기다.
- 중국 방면 상대는 청이며, 명나라 표현은 사용하지 않는다.
- `sinuiju` 데이터 ID는 저장 호환성을 위해 유지하지만 화면 표시명은 `의주`다.
- `chongjin` 데이터 ID는 저장 호환성을 위해 유지하지만 화면 표시명은 `경흥`이다.
- `qing_yalu_market`은 압록강 너머 청 상인을 만나는 `책문 장시`다.
- `tumen_north_market`은 두만강 너머 북방 상인을 만나는 `두만강 북방장`이다.
- 두 국경 장시는 허가장과 큰 수레/호위 준비가 필요한 장기 목표이며, 중국/러시아 본토 확장이 아니다.
- 상세 특산품 표와 지도 기준은 `docs/WORLDBUILDING_TRADE_AUDIT.md`를 따른다.

## 21. 새 작업자가 반드시 지킬 것

- localStorage 저장 키를 함부로 바꾸지 말 것.
- 데이터 ID를 함부로 바꾸지 말 것.
- 중국 본토, 일본 본토, 항구 투자, 실시간 전투를 추가하지 말 것.
- 교역/이동/저장/의뢰 루프를 깨지 말 것.
- 화면은 가로형 모바일 기준으로 검증할 것.
- UI는 텍스트보다 그림, 아이콘, 핫스팟, 짧은 CTA 중심으로 유지할 것.
- 기능 추가보다 “아이와 함께 보고 누를 수 있는 게임성”을 우선할 것.
- 변경 후 `validate:data`, `audit:consistency`, `build`, `smoke`, 가능하면 `visual`을 실행할 것.

## 22. 2026-05-04 현재 UI/그래픽 기준

- 본게임은 가로형 full mode를 기준으로 한다. 휴대폰 세로 화면에서는 게임 화면을 축소하거나 아래에 숨겨 렌더링하지 않고, 회전 안내 화면만 보여준다.
- 세로 안내 화면 문구는 "이 게임은 가로 화면에서 더 잘 보여요. 휴대폰을 옆으로 돌려주세요."이며, "계속" 버튼은 설명 확인용이다.
- 모바일 landscape의 기준 해상도는 `844x390`, `932x430`, `1024x600`이다. HUD, 탭, 지도, 우측 패널, 하단 안내가 서로 겹치지 않아야 한다.
- 현재 그래픽 기준은 `painted2d` 자산 중심의 2D 게임 화면이다. 인라인 polygon 지도, 작은 hub SVG, 그림판식 이동 토큰, 223-byte placeholder 결과 아이콘은 primary UI에서 사용하지 않는다.
- 전체 지도는 `assets/painted2d/maps/korea-route-map.webp` 위에 노드/루트/라벨을 얹는 방식이며, 두만강/압록강과 국경 장시 라벨은 UI 레이어로 계속 유지한다.
- 상품, 시설, 장비, NPC, 요정 이미지는 찌그러지거나 잘리지 않도록 `object-fit`과 프레임 규칙을 통일한다.
