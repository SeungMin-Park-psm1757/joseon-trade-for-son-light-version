# 플레이테스트 수정 프롬프트

아래 플레이테스트 피드백을 바탕으로 앱을 수정해 주세요.

## 수정 원칙

- 교역 루프와 모바일 조작성을 최우선으로 유지하세요.
- 새 기능을 크게 늘리기보다 불편함과 버그를 먼저 고치세요.
- 데이터 값 조정은 `public/data/*.json`에서 처리하세요.
- 코드에 임시 하드코딩을 추가하지 마세요.
- 수정 후 `npm run build`를 실행하고 결과를 보고하세요.

## 피드백 입력란

여기에 플레이테스트 결과를 붙여넣으세요.

```text
- 문제 1:
- 문제 2:
- 개선 희망:
```

## 보고 형식

- 수정한 문제
- 수정하지 못한 문제
- 추가 확인이 필요한 부분
- 다음 추천 작업
