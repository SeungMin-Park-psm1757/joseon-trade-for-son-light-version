# 모바일 UI 개선 프롬프트 — 작은 변경 반복용

현재 앱의 기능은 유지하면서 모바일 사용성을 개선해 주세요. 한 번에 큰 리디자인을 하지 말고, 아래 항목 중 **가장 중요한 1~2개 표면만** 먼저 수정하세요.

## 목표

- 390px 폭 세로 화면에서 조작하기 쉬운 UI
- 큰 버튼과 읽기 쉬운 글자
- 현재 돈/날짜/위치가 항상 보이는 상태바
- 하단 탭 고정
- 지도와 카드의 스크롤 충돌 최소화
- 시장 구매/판매 버튼의 오작동 방지

## 우선 개선 후보

1. 버튼 높이 44px 이상
2. 주요 숫자 강조: 돈, 가격, 화물칸, 위험도
3. 위험 태그 배지화
4. 수익/손실 피드백 명확화
5. 항구 카드 간격 조정
6. 이벤트 모달 가독성 개선
7. 작은 화면에서 가로 스크롤 제거
8. iOS safe-area 고려

## 유지해야 할 것

- 기존 데이터 구조
- LocalStorage 저장 키
- 교역/이동/이벤트 로직
- 한국어 UI
- 기존 `data-testid`

## 검증

- dev server를 실행하거나 재사용하세요.
- 390 × 844 viewport에서 변경 화면을 확인하세요.
- 가능하면 `npm run build`와 `npm run test:smoke`를 실행하세요.
- 변경 파일과 브라우저 확인 결과를 보고하세요.
