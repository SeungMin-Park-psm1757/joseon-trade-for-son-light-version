# 브라우저 플레이테스트 프롬프트

앱을 실제 브라우저에서 실행하고 모바일 viewport 기준으로 플레이테스트해 주세요.

## 실행

```bash
cd starter
npm install
npm run dev
```

가능하면 Playwright 또는 Codex의 브라우저 도구를 사용하세요.

## viewport

- 기본: 390 × 844
- 추가: 360 × 740, 430 × 932

## 테스트 흐름

```text
새 게임 시작
→ 부산포 상태 확인
→ 시장에서 면포 2개 또는 건어물 2개 구매
→ 지도에서 대구/통영/여수 중 연결 루트 선택
→ 이동 전 위험·소요일 확인
→ 이동 실행
→ 이벤트 발생 시 선택지 처리
→ 도착지 시장에서 판매
→ 손익 로그 확인
→ 새로고침
→ 이어하기로 상태 복구 확인
```

## 확인할 것

- 교역 루프가 끊기지 않는가?
- 이벤트가 이해 가능한가?
- 돈/화물/날짜가 정확히 변하는가?
- 모바일에서 버튼이 누르기 쉬운가?
- 가로 스크롤이나 화면 밖 모달이 없는가?
- 자동 저장이 실제로 되는가?

## 결과 처리

- 발견한 버그는 우선순위별로 정리하세요.
- 치명적 버그는 바로 수정하세요.
- UI 취향 문제는 별도 목록으로 남기세요.
- `.logs/`에 플레이테스트 로그를 저장하세요.
