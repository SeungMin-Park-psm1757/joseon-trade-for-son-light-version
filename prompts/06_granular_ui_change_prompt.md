# 작은 UI 변경 요청 템플릿

아래 템플릿의 빈칸을 채워 Codex에 전달하세요. 이 프롬프트는 기능이 이미 작동하는 앱에서 UI만 빠르게 다듬을 때 사용합니다.

```text
다음 UI 변경 하나만 처리해 주세요.

대상 화면/surface:
- 예: 시장 화면 / 지도 화면 / 이벤트 모달 / 하단 탭

현재 문제:
- 예: 390px 폭에서 구매 버튼과 판매 버튼이 너무 붙어 있어 실수로 누르기 쉽다.

목표 변경:
- 예: 상품 카드 안의 구매/판매 버튼 간격을 8px 이상으로 늘리고, 버튼 높이를 최소 44px로 맞춰라.

검증 viewport:
- 390 × 844 모바일 세로 화면

제약:
- 필요한 파일만 최소 수정한다.
- 기존 컴포넌트, 상태 흐름, 데이터 구조, 저장 키를 보존한다.
- 게임 로직과 가격 계산은 바꾸지 않는다.
- 기존 data-testid를 제거하지 않는다.
- dev server를 실행하거나 재사용해서 브라우저에서 결과를 확인한다.
- 이 변경 하나를 끝내면 멈추고, 변경 파일과 브라우저 확인 내용을 요약한다.
```

## 후속 수정용 문장

```text
방금 변경은 거의 맞습니다. 구현은 유지하되 이 세부사항만 조정해 주세요:
- [남은 불일치]
같은 화면과 같은 viewport에서 다시 확인한 뒤 멈추세요.
```
