# Codex 자가검토 프롬프트 — 빌드·데이터·브라우저 확인

방금 구현한 앱을 다음 기준으로 자가검토하고 필요한 수정을 해 주세요.

## 1. 명령 검증

```bash
cd starter
npm run validate:data
npm run build
npm run test:smoke
```

실패하면 원인을 고치고 다시 실행하세요.

## 2. 기능 검토

1. 새 게임을 시작할 수 있는가?
2. 이어하기가 LocalStorage에서 정상 작동하는가?
3. 부산포에서 시작하는가?
4. 시장에서 상품을 구매/판매할 수 있는가?
5. 화물칸 제한이 적용되는가?
6. 현재 위치에서 연결된 루트만 표시되는가?
7. 이동하면 날짜가 경과하는가?
8. 이동 중 이벤트가 발생할 수 있는가?
9. 월이 바뀌면 가격표가 바뀌는가?
10. 같은 달에는 가격표가 불필요하게 계속 변하지 않는가?
11. 어업은 하루를 소모하고 보상을 주는가?
12. 대마도 루트는 허가장 조건을 확인하는가?
13. 배/수레 구매가 가능하고 화물칸 계산에 반영되는가?
14. 퀘스트 3개 이상이 보이고 완료 보상이 적용되는가?
15. 데이터 JSON을 수정하면 앱 내용이 바뀌는가?

## 3. 모바일 브라우저 검토

가능하면 Playwright 또는 브라우저 도구로 390 × 844 viewport에서 확인하세요.

- 가로 스크롤이 생기지 않는가?
- 버튼이 손가락으로 누르기 쉬운가?
- 상단 상태바에서 돈/날짜/위치가 보이는가?
- 하단 탭이 safe-area에 가리지 않는가?
- 이벤트 모달이 화면 밖으로 넘치지 않는가?
- 시장 수량 조절이 실수하기 어렵게 되어 있는가?

## 4. 수정 방식

- 발견한 문제를 먼저 목록화하세요.
- 즉시 수정 가능한 것은 고치세요.
- 구조적 리팩터링이 필요한 경우 최소 범위로 정리하세요.
- UI만 고치는 작업이면 게임 로직을 건드리지 마세요.
- 완료 후 `.logs/`에 자가검토 로그를 남기세요.

## 5. 보고 형식

- 발견한 문제
- 수정한 문제
- 통과한 검증 명령
- 브라우저 확인 결과
- 남은 이슈
